# Agent guide for OnlyCLI

This document is for AI coding assistants (Cursor, GitHub Copilot, OpenAI Codex, Claude, and similar tools) working on the OnlyCLI repository.

## Project overview

OnlyCLI is a Go program that reads an OpenAPI specification and emits a **native Go CLI project**: Cobra commands, an HTTP client, and supporting code. Parsing uses **libopenapi**, the CLI framework is **Cobra**, and output is produced with **Go’s `text/template`** plus `go/format` for emitted Go sources.

## Module path

```
github.com/onlycli/onlycli
```

## Key commands

| Command       | Purpose                                      |
|---------------|----------------------------------------------|
| `make build`  | Build the `onlycli` binary into `bin/`.      |
| `make test`   | Run unit tests with race detector and cover. |
| `make check`  | Run `fmt`, `vet`, `lint`, and `test`.        |
| `make lint`   | Run `golangci-lint` on the module.           |

Run `make check` before opening a PR or committing changes that touch code.

## Project layout

| Path | Role |
|------|------|
| `cmd/onlycli/` | CLI entry point (`main`), Cobra root and subcommands. |
| `internal/model/` | Intermediate representation (IR) structs and naming helpers. |
| `internal/parser/` | OpenAPI loading and mapping into the IR (libopenapi). |
| `internal/codegen/` | Template-driven code generation engine. |
| `internal/codegen/templates/` | Embedded Go template sources (`*.tmpl`). |
| `internal/testdata/` | Fixtures for parser and generator tests. |
| `integration_test.go` | End-to-end integration tests at repository root. |
| `examples/` | Example CLI projects produced by `make generate-example` (e.g. `examples/petstore`). |

## Conventions

- **Generated file header:** All code emitted by OnlyCLI must begin with:
  - `// Code generated by onlycli; DO NOT EDIT.`
- **Formatting:** Use `go/format` (`format.Source`) on generated Go files, not ad-hoc formatting.
- **Tests:** Keep **unit tests** next to the package they cover (`*_test.go` under `internal/...`). Keep **integration tests** in `integration_test.go` at the repo root.
- **Pre-commit quality bar:** Run `make check` before committing.
- **Commit messages:** Prefer [Conventional Commits](https://www.conventionalcommits.org/): `feat:`, `fix:`, `docs:`, `test:`, `chore:`.

## Dependencies (onlycli itself)

- **libopenapi** — parse and traverse OpenAPI 3.x documents.
- **cobra** — CLI for the `onlycli` tool and the pattern used in generated projects.
- **text/template** — codegen; templates live under `internal/codegen/templates/` and are embedded with `go:embed`.

## Test data

| File | Use |
|------|-----|
| `internal/testdata/petstore.yaml` | Small, familiar spec for basic behavior. |
| `internal/testdata/github_subset.yaml` | Larger, more realistic surface for integration-style coverage. |

When adding behavior, extend or add fixtures under `internal/testdata/` and cover the parser/codegen paths with tests.
