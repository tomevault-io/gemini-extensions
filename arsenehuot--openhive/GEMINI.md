# OpenHive

Open-source team messaging platform — Next.js webapp with Supabase backend.

## Architecture

- **Frontend**: Next.js 14+ (App Router), TypeScript, shadcn/ui + Tailwind CSS
- **Backend**: Supabase (PostgreSQL, Auth, Realtime, Edge Functions, Storage)
- **State**: Zustand
- **Calls**: LiveKit WebRTC (Phase 2)
- **Config**: `.env.local` (gitignored) — generated by setup wizard or manual

## Project Structure

```
src/
├── app/                    # Next.js App Router
│   ├── api/setup/         # POST: writes .env.local (dev only)
│   ├── setup/             # Setup wizard (connect Supabase)
│   ├── auth/              # Sign in / Sign up
│   └── workspace/         # Main app (sidebar + channels + messages)
├── components/
│   ├── ui/                # shadcn/ui components
│   ├── sidebar/           # Workspace sidebar
│   ├── chat/              # Messages, input, threads
│   └── workspace/         # Workspace creation
├── lib/
│   ├── supabase/          # Client, provisioner, migrations (23 tables)
│   └── store/             # Zustand stores
├── hooks/                 # React hooks
└── types/                 # TypeScript types for all tables
```

## Database

All 23 tables are auto-provisioned via the setup wizard using the Supabase Management API.
RLS policies use SECURITY DEFINER helper functions to avoid recursion.
Realtime is enabled on messages, channels, calls, profiles, reactions.

## Getting Started

```bash
npm install
npm run dev
# Visit http://localhost:3000 → Setup wizard
```

## Key Config

- NO hardcoded Supabase config in code
- `.env.local` is gitignored
- Setup wizard writes `.env.local` via `/api/setup`
- For production: set env vars in deployment platform (Vercel, etc.)

## Key Dependencies

- @supabase/supabase-js
- zustand
- lucide-react
- date-fns
- shadcn/ui (components in src/components/ui/)
