# Threelab

Generative art platform: React 19 + Three.js frontend, Go + MongoDB backend.

## Ports
- Frontend dev: 4911 (Vite, proxies /api and /mcp to backend)
- Backend: 4912

## Build check
```bash
cd frontend && npx tsc --noEmit
cd backend && go build ./...
```

## Project layout
- `frontend/src/patterns/PatternRegistry.ts` — central pattern type registry (21 built-ins + user patterns)
- `frontend/src/nodes/` — node graph system (types, executor, storage, factories)
- `frontend/src/components/NodeEditor/` — visual graph editor + code editor
- `frontend/src/utils/mutateGenome.ts` — client-side evolution engine
- `backend/mcp/server.go` — MCP server (13 tools)
- `backend/services/evolution.go` — server-side mutation (used by MCP/API)
- `backend/services/schemas.go` — pattern parameter schemas
- `backend/services/export.go` — HTML/React/JSON export templates

## Key conventions
- User patterns stored in localStorage, keyed `threelab_user_patterns`
- Built-in patterns are read-only node graphs generated by `generatorGraphFactory.ts`
- Pattern types prefixed `user_builtin_` are factory-generated built-ins; `user_` prefix for user-created
- Backend strips `user_builtin_` and `user_` prefixes when looking up schemas
- Anonymous auth uses cookie-based UID; `Secure` flag is conditional on HTTPS
- Client-side evolution in `mutateGenome.ts`; server-side in `services/evolution.go` (for MCP/API)
- NEVER drop or clear the database autonomously
