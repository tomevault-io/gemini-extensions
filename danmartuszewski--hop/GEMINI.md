# Commit Messages

Use conventional commit format with imperative mood:

```
type: short description
```

Types: `feat`, `fix`, `docs`, `refactor`, `chore`, `test`, `ci`

Examples:
- `feat: add tag filter UI to dashboard`
- `fix: improve connection form UX`
- `docs: sync documentation with recent feature additions`
- `refactor: remove connection details panel from dashboard`
- `chore: remove outdated specs directory`
