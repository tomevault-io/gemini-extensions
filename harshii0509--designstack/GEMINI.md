## Four principles (always apply these)

1. **Think in references, not descriptions.** If the user says "make it look better", ask for a reference or specific element — don't guess what "better" means.
2. **Summarize before changing.** Before editing any UI or CSS file, write one sentence saying what you're about to change and why. Wait for confirmation.
3. **Read the Bible before touching visuals.** If `design/DESIGN-BIBLE.md` exists in this project, read it before making any visual, layout, or style change. Apply its color, font, spacing, and component rules.
4. **Flag drift, don't silently fix.** If you notice a value (color, font size, spacing) that doesn't match the Design Bible, say so before changing it.
