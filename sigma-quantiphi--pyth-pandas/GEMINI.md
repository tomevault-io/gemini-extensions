## Architecture (mirrors polymarket-pandas conventions)

- **Sync first, async via composition.** `AsyncPythPandas` wraps a sync
  `PythPandas` and runs each method in a `ThreadPoolExecutor`. Wrappers are
  auto-generated by `_populate_async_methods()` using `callable()` so
  `cachedmethod` descriptors are caught.
- **Every DataFrame return has a pandera schema** (`ParsedFeedSchema`) and a
  `DataFrame[Schema]` annotation. `_Lenient` base: `strict=False`,
  `coerce=True`.
- **Every dict return has a `TypedDict`** in `types.py`.
- **Auth guards** raise `PythAuthError` *before* any HTTP call when the
  bearer token is missing.
- **Single error mapping**: `_handle_response` is the only place HTTP status
  codes turn into exceptions.
- **`preprocess_dataframe` pipeline**: snakeâ†’camel, drop columns, coerce
  numeric/datetime/bool/json by name. Pyth uses microsecond timestamps
  pervasively, so this fork adds a dedicated `us_int_datetime_columns` axis
  beyond the four standard ones.

## Adding endpoints

1. Add a method to the appropriate mixin in `mixins/_<tag>.py`.
2. If it returns a list, define / extend a pandera schema in `schemas.py`
   and use `DataFrame[Schema]` as the return type.
3. If it returns a dict, define a `TypedDict` in `types.py`.
4. Re-export new schemas/TypedDicts from `__init__.py`.
5. Add a mocked unit test in `tests/test_unit.py` (and async mirror).
