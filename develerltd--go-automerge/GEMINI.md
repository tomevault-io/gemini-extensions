# go-automerge

Pure Go implementation of [automerge](https://github.com/automerge/automerge).
Binary-compatible with the Rust implementation -- documents produced by either can be loaded by the other.

## Upstream Tracking

- **Upstream repo:** https://github.com/automerge/automerge
- **Tracked commit:** `8246d0f8218cd49dc1af56ba6eefd88ed215665c`
- **Tracked version:** `automerge 0.8.0`
- **Go module version:** `v0.8.0`

To check for upstream changes since last sync:
```bash
git -C /path/to/automerge log --oneline 8246d0f82..HEAD -- rust/automerge/ rust/hexane/
```

## Architecture Mapping

| Rust crate / module         | Go package           |
|-----------------------------|----------------------|
| `automerge` (top-level API) | `automerge`          |
| `automerge::op_set`         | `internal/opset`     |
| `automerge::columnar`       | `internal/columnar`  |
| `automerge::storage`        | `internal/storage`   |
| `automerge::types`          | `internal/types`     |
| `automerge::sync`           | `internal/sync`      |
| `automerge::encoding`       | `internal/encoding`  |
| `hexane`                    | `internal/hexane`    |

## Development

```bash
# Run all tests
go test -race ./...

# Run cross-implementation tests only
go test ./automerge/ -run TestCrossImpl -v

# Run Rust CLI verification (requires built automerge-cli)
AUTOMERGE_CLI=/path/to/automerge/rust/target/release/automerge go test ./automerge/ -run TestCrossImplRustCLI -v

# Or if automerge repo is at ../../automerge relative to this repo:
go test ./automerge/ -run TestCrossImplRustCLI -v
```

## Cross-Implementation Testing

Test fixtures in `automerge/testdata/` are generated by `fixture-gen` in the upstream repo.

To regenerate fixtures:
```bash
cd /path/to/automerge/rust
cargo run --manifest-path fixture-gen/Cargo.toml -- /path/to/go-automerge/automerge/testdata
```

## Conventions

- Public API lives in the `automerge` package
- Internal packages under `internal/` are not importable externally
- Error sentinels defined in `automerge/errors.go`
- Cross-impl test functions prefixed `TestCrossImpl`
- Test files follow Go convention: `*_test.go` in the same package
