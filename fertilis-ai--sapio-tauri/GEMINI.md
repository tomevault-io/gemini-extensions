## Project Overview

Sapio-app is a modern TypeScript monorepo for building a consumer AI agent application. The project uses Better-T-Stack and combines React 19, TanStack Router, TailwindCSS, shadcn/ui, and Tauri to create both web and native desktop applications from a single codebase.

**Vision:** This project implements the specification in SPEC.md - a next-generation personal AI assistant that lives on user devices, learns over time, acts autonomously, and keeps data private through a local-first architecture.

## Common Commands

### Development
```bash
# Start all applications in development mode
bun run dev

# Start only the web application (runs on http://localhost:3001)
bun run dev:web

# Start Tauri desktop application in development mode
cd apps/web && bun run desktop:dev
```

### Building
```bash
# Build all applications for production
bun run build

# Build Tauri desktop application
cd apps/web && bun run desktop:build
```

### Type Checking
```bash
# Check TypeScript types across all workspaces
bun run check-types

# Check types for web app only
cd apps/web && bun run check-types
```

### Package Management
```bash
# Install dependencies (run from root)
bun install

# Add a new dependency to web app
cd apps/web && bun add <package-name>

# Add a shared dependency using workspace catalog
# 1. Add to root package.json workspaces.catalog
# 2. Reference with "catalog:" in app package.json
```

## Architecture

### Monorepo Structure

This is a **Turborepo monorepo** with Bun as the package manager. The architecture follows these principles:

1. **Workspace Catalog**: Shared dependency versions defined in root `package.json` under `workspaces.catalog` (dotenv, zod, typescript)
2. **Local Packages**: Internal packages referenced via `workspace:*` protocol
3. **Build Orchestration**: Turborepo manages task dependencies and caching
4. **Shared Configuration**: TypeScript and build configs centralized in `@sapio-app/config`
5. **App logs**: `~/.sapio/logs/`

```
apps/
  web/              # Main React application + Tauri desktop runtime
    src/            # React source code
    src-tauri/      # Tauri Rust backend for desktop app
packages/
  config/           # Shared TypeScript configuration
  env/              # Environment variable management (T3 env-core)
```

### Technology Stack

**Desktop Runtime:**

- Rust backend (in src-tauri/)

**Build System:**
- Bun v1.3.6 - Package manager and runtime
- Turborepo v2.6.3 - Monorepo orchestration
- Vite - Frontend bundler with HMR

### Key Architectural Patterns

#### 1. File-Based Routing
Routes are automatically generated from the file structure in `apps/web/src/routes/`. The TanStack Router plugin generates `routeTree.gen.ts` automatically - **do not edit this file manually**.

Example:
- `src/routes/index.tsx` → `/` route
- `src/routes/about.tsx` → `/about` route
- `src/routes/users/$id.tsx` → `/users/:id` route

#### 2. Path Aliases
All imports from `src/` can use the `@/` alias:
```typescript
import { cn } from "@/lib/utils"
import { Button } from "@/components/ui/button"
```

#### 3. Component Organization
- `src/components/ui/` - shadcn/ui components (pre-built, customizable)
- `src/components/` - Custom application components
- `src/lib/utils.ts` - Shared utility functions (e.g., `cn()` for class merging)

#### 4. Styling Conventions
- Use TailwindCSS utility classes
- Use `cn()` utility to merge classes intelligently
- Theme variables defined as CSS custom properties
- Components use `class-variance-authority` for variant management

#### 5. Environment Variables
Centralized in `packages/env/src/web.ts` using T3's env-core for type-safe environment validation. Currently minimal - expand as needed with schema validation:

```typescript
export const env = createEnv({
  server: {
    DATABASE_URL: z.string().url(),
  },
  client: {
    PUBLIC_API_URL: z.string().url(),
  },
  runtimeEnv: process.env,
})
```

#### 6. Tauri Integration
The Tauri desktop app configuration lives in `apps/web/src-tauri/`:
- `tauri.conf.json` - App metadata, window config, build settings
- `src/lib.rs` - Tauri initialization and plugin setup
- `src/main.rs` - Application entry point
- `Cargo.toml` - Rust dependencies

Dev mode runs frontend on `http://localhost:3001` and Tauri connects to it. Production builds bundle the frontend dist into the native app.

#### 7. Build Process
Turborepo orchestrates builds with dependency awareness:
```
turbo build
  ├── @sapio-app/config (no build output)
  └── @sapio-app/env (no build output)
    └── web (depends on ^build)
       ├── Vite builds React → dist/
       └── Tauri packages desktop app (production only)
```

The `^build` dependency in turbo.json ensures packages build before apps that depend on them.

### Important Configuration Files

- `turbo.json` - Task orchestration, caching strategy, environment variable handling
- `apps/web/vite.config.ts` - Vite plugins (Tailwind, TanStack Router, React), path aliases, dev server port
- `apps/web/components.json` - shadcn/ui configuration (import paths, styling)
- `apps/web/src-tauri/tauri.conf.json` - Desktop app configuration
- `packages/config/tsconfig.base.json` - Base TypeScript config extended by all workspaces

### Type Safety

The project enforces strict TypeScript:
- `strict: true` in base config
- `noUnusedLocals: true` and `noUnusedParameters: true`
- `verbatimModuleSyntax: true` for explicit import types
- All workspaces extend shared base config

## Project Context

This codebase implements the vision described in `SPEC.md`:

**Core Design Principles:**
1. **Reliability First** - Agent must be trustworthy
2. **Privacy by Default** - Local-first architecture, user data stays on-device
3. **Progressive Autonomy** - Starts helpful, earns trust, gains independence
4. **Multi-Agent Native** - Built for specialized agents to collaborate
5. **Consumer-Grade UX** - Simple for non-technical users

**Architecture Alignment:**
- **Local-first**: Tauri enables on-device processing (no cloud required)
- **Privacy**: Desktop app runs completely local, E2E encryption for optional sync
- **Memory System**: Will be implemented in local file-based DB (specified in SPEC.md Section 2)
- **Perception Layer**: Will integrate with system APIs via Tauri commands
- **Action Layer**: Tauri enables OS integration, file system access, API calls

**Implementation Status:**
This is a fresh project (initial commit). The foundation is established with modern tooling. Future work will implement the agent core, memory system, autonomy framework, and multi-agent coordination as specified in SPEC.md Sections 5-6.

## Development Workflow

1. **Adding Routes**: Create files in `src/routes/`, TanStack Router auto-generates route tree
2. **Adding UI Components**: Use shadcn CLI or copy from shadcn/ui docs into `src/components/ui/`
3. **Adding Tauri Commands**: Define Rust functions in `src-tauri/src/lib.rs`, invoke from React via `@tauri-apps/api`
4. **Environment Variables**: Add schema to `packages/env/src/web.ts`, use typed `env` object
5. **Styling**: Use Tailwind classes, theme variables from `app.css`, `cn()` for merging
