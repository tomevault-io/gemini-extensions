# A Guide to Accounting for Reporting Delays in State, Tribal, Local, and Territorial Public Health Surveillance Data

## Project structure

- `manuscript/` - Content plan and drafts for the preprint/journal submission
- `user-submitted-snippets/` - Code snippets and notebooks from users (future)
- `guidance-website/` - Guidance website content (future, coordination centre)
- `meeting/` - Dissemination and meeting planning
- `resources/` - Reference materials and archived plans

## Audiences

- Epidemiologists and public health practitioners interested in accounting for reporting delays and other real-time data issues
- Modellers expanding their understanding of reporting delays and other real-time data issues

## Writing conventions

- UK English throughout
- One sentence per line in markdown
- Simple, direct prose without unnecessary adjectives
- Use `@placeholder` for missing references

## Technical conventions

- Quarto: Use `.qmd` extension for notebooks
- Use `task` to run common commands (see `Taskfile.yml`)
- Max 80 characters per line
