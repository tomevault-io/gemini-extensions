# AGENTS.md

## Architecture intent

- `App` = user-facing, has a canvas and runs in its own FreeRTOS task
- `Service` = background task, no canvas, runs indefinitely
- Both inherit `KeiraThread` — don't bypass this hierarchy
- `ksystem` is the global entry point to AppManager, ServiceManager, etc.

## Non-obvious conventions

- `K_BTN_*` defines (`keira.h`) are `lilka::Button` enum values — use them for UI components (`menu.addActivationButton(K_BTN_BACK)`, `menu.getButton() == K_BTN_EXIT`), not for reading raw button state
- To read button state, use named fields on `lilka::State` directly: `state.a`, `state.b`, `state.up`, `state.start`, etc. — `State` has no array indexing
- Use `K_FT_*_HANDLER(X)` macros (`keira.h`) for file type dispatch
- Use `StringFormat()` instead of `sprintf()`
- Use `#pragma once` — never `#ifndef` include guards
- Use `defer` (`src/utils/defer.h`) for scope-exit cleanup
- Use `acquire` (`src/utils/acquire.h`) for RAII mutex locking
- Use `KMTX_SETER_GETER` macro for thread-safe getters/setters on shared state

## What not to touch

- `src/keira/keira_version_auto_gen.h` — auto-generated at build time
- `.pio/` — generated by PlatformIO
- `lib/mJS/` and `lib/SimpleFTPServer/` — vendored, don't reformat

## Build gotchas

- The [Lilka SDK](https://github.com/lilka-dev/sdk) must be checked out as a **sibling directory** to `keira/` — `lib_dir = ../../sdk/lib` in `platformio.ini`
- **v2** (ESP32-S3) is the primary target — default in PlatformIO and the one to test against
- **v1** (ESP32-C3) is legacy — only touch if explicitly asked
- `clang-format` and `cppcheck` are enforced in CI — run `make clang-format` and `make cppcheck` before committing

## Arduino vs esp-idf

- `String` (Arduino) is the only Arduino type allowed — use it for all string handling
- For everything else prefer esp-idf equivalents over Arduino wrappers:
  - `xTaskCreate` / `vTaskDelay` / `SemaphoreHandle_t` over Arduino task abstractions
  - `esp_wifi_*` over `WiFi.*`
  - `esp_http_client_*` over `HTTPClient`
- Use `vTaskDelay()` instead of `delay()`
- `Preferences` is acceptable temporarily until a better storage mechanism is introduced
- `pinMode` / `digitalWrite` are acceptable temporarily until a GPIO abstraction is introduced
- When esp-idf and Arduino both work, esp-idf is preferred

## Lilka SDK

See [Lilka SDK](https://github.com/lilka-dev/sdk) and its `AGENTS.md` for SDK-specific conventions.

## Filesystem

- Always use C-style file API (`fopen`, `fread`, `fwrite`, `fclose`) with canonical paths (`/sd/...`, `/spiffs/...`) — never use `SD.open()`, `SPIFFS.open()`, or any Arduino filesystem wrappers

## Global state

- No global variables in `src/apps/` or `src/services/` implementations
- All state must be members of the `App` or `Service` subclass
- File-scoped constants (`static const` / `constexpr`) are fine — mutable globals are not
