# HypGo — GitHub Copilot Instructions

Full rules: `config/AI_CODING_RULES.md`

## Critical rules (must follow)

1. **Every func requires `@ai` comment block**. Exported funcs require: `@ai:generated`, `@ai:purpose`, `@ai:input`, `@ai:output`, `@ai:sideeffect`.
2. **Date format**: `YYYY-MM-DD` only. Model name: official identifier (e.g. `copilot`, `claude-sonnet-4-6`).
3. **Schema-first routing**: Use `r.Schema(...)` for REST, `schema.RegisterGRPC(...)` for gRPC. Never write raw Route structs.
4. **Error handling**: Use `pkg/errors` typed catalog + `errors.AbortWithAppError(c, ...)`. Never `c.JSON(500, ...)` directly.
5. **DB access**: `hidb.ReadHypDB()` for reads (replica), `hidb.WriteHypDB()` for writes (master).

## Func comment template

```go
// FuncName <verb-phrase describing what it does>.
//
// @ai:generated by=copilot date=YYYY-MM-DD
// @ai:purpose <why this func exists, not what it does>
// @ai:input <param semantics, not types>
// @ai:output <return semantics + status codes>
// @ai:sideeffect <DB/IO/network or "none">
func FuncName(...) {}
```

## Forbidden

- `panic()` in handlers
- `fmt.Println` for logging (use `pkg/logger`)
- `math/rand` for tokens (use `crypto/rand`)
- Hardcoded DSN / API key / secrets

Read `config/AI_CODING_RULES.md` for full specification.
