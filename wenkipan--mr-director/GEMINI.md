## mr-director

## Design Context

### Users
- **独立创作者/UP主**：需要快速把录屏、素材变成抖音/B站短视频，不想学复杂的剪辑软件，用自然语言描述意图即可
- **技术爱好者/开发者**：对 AI + 视频技术感兴趣的极客，享受本地部署、自定义配置的乐趣，会阅读代码和修改 prompt
- **使用场景**：用户通常独自操作，在自己的电脑上本地运行，通过聊天窗口与 AI Agent 协作完成剪辑

### Brand Personality
- **轻松** — 不板着脸的工具，降低视频剪辑的心理门槛
- **智能** — AI-first 体验，Agent 主动理解意图、提出方案、等待确认
- **亲切** — 像一个懂技术的朋友在帮你干活，而不是一个冷冰冰的专业软件
- **语气**：随和但不幼稚，技术但不炫技，中文为主交互语言

### Aesthetic Direction
- **视觉基调**：类 VS Code / 终端风格 — 深色、紧凑、信息密度高
- **主题**：纯深色模式（Dark Only），zinc 色系为底色
- **配色**：zinc-950 背景 / zinc-900 面板 / zinc-800 边框 / blue-600 主强调色
- **技术栈**：Tailwind CSS 3（当前为默认配置，无自定义 token）
- **参考**：VS Code、终端工具的信息密度和效率感
- **反面案例**：避免过于圆润/低信息密度的大众消费级 UI（如 CapCut/剪映），避免过度装饰性的视觉元素

### Design Principles

1. **对话驱动，工具辅助** — 聊天面板是核心交互入口，时间线和预览是 Agent 工作的可视化结果。设计应让聊天体验流畅，同时确保时间线编辑区域功能可达
2. **信息密度优先** — 屏幕空间宝贵（三栏布局），每个像素都应传递有用信息。紧凑排布，减少无意义留白，但通过清晰的层级和边界保持可读性
3. **渐进展示复杂度** — 默认展示核心信息（聊天、预览、时间线），高级控件（属性编辑器、导出选项）按需展开，不让新手被功能淹没
4. **状态透明** — Agent 的思考过程、工具调用状态、WebSocket 连接状态应始终清晰可见，让用户知道系统在干什么
5. **WCAG AA 合规** — 确保对比度、键盘可操作性、焦点指示器等满足 WCAG 2.1 AA 标准，尤其注意深色主题下的文字可读性
