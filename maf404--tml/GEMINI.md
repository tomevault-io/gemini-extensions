# OATR Test Suite
**Project:** Open Agent Tool Registry (OATR)
**Author:** Matias Fernandez
**Purpose:** Validate TML and .min formats against real APIs, comparing token efficiency and accuracy vs existing schema formats

---

## Hypothesis

> TML and .min formats can describe real API tools with enough fidelity for an AI agent to correctly call them — using significantly fewer tokens than OpenAPI JSON, MCP JSON-RPC, and other existing formats.

We test **two things**:
1. **Tool selection** — given a natural language task, does the agent pick the right tool?
2. **Tool execution** — given the schema, does the agent construct a valid API call that returns real data?

## Tests

### 1. `test.js` — Original TML vs .min

Compares TML Source and .min Index against 3 weather scenarios using Open-Meteo API (no auth).

```bash
node test.js
# Outputs: report.md
```

**Last results:** Both formats 100% accuracy. .min uses ~53% fewer tokens than TML.

---

### 2. `benchmark.js` — 7-Format Comparison

Tests 7 formats × 5 scenarios = 35 API calls. All use the same 3 Open-Meteo weather tools.

| Format | Group | What it tests |
|--------|-------|---------------|
| Plain English Prose | baseline | Can natural language match structured formats? |
| TypeScript Signatures | existing | Developer-friendly, strongly typed |
| Minimal JSON | existing | Stripped JSON, no descriptions |
| OpenAI JSON Schema | existing | Function calling format used by OpenAI |
| TML Source | oatr | Our format — human + AI optimized |
| .min Index | oatr | Our compiled bootstrap format |
| OpenAPI JSON | existing | Industry standard, verbose |

**5 scenarios:**
1. Current weather — known US city (Melbourne FL)
2. Rain forecast — international city (Buenos Aires)
3. Current temp — international city (Tokyo)
4. Implicit planning intent — "picnic on Saturday" (Miami)
5. Current weather — uncommon city (Reykjavik)

```bash
node benchmark.js
# Outputs: benchmark-report.md
```

**Last results:**
| Format | Accuracy | Tokens | vs OpenAPI |
|--------|----------|--------|------------|
| .min Index | 100% | 174 | -77% |
| Minimal JSON | 100% | 251 | -67% |
| Plain English | 100% | 364 | -53% |
| TypeScript | 100% | 364 | -53% |
| TML Source | 100% | 368 | -52% |
| OpenAI Schema | 100% | 522 | -32% |
| OpenAPI JSON | **60%** | 769 | baseline |

OpenAPI failed scenarios 2 and 4 (rain/planning queries) — its verbose single-endpoint spec confused tool disambiguation.

### 3. `mcp-compare.js` — MCP JSON-RPC vs OATR

Direct comparison of MCP's tools/list JSON-RPC schema format against TML and .min. 3 formats × 5 scenarios = 15 calls.

```bash
node mcp-compare.js
# Outputs: mcp-compare-report.md
```

**Last results:**
| Format | Accuracy | Tokens (3 tools) | vs MCP |
|--------|----------|-------------------|--------|
| MCP JSON-RPC | 100% | 765 | baseline |
| TML Source | 100% | 320 | -58% |
| .min Index | 100% | 165 | -78% |

All formats achieved 100% accuracy. .min delivers identical results at 78% fewer tokens than MCP.

## Known Issues

1. **spike.js `normalizeId` bug** — `normalizeId()` at line ~204 replaces ALL underscores with dots. When Claude returns `countries_by_name`, it becomes `countries.by.name` instead of `countries.by_name`. This breaks tool lookup for REST Countries. Fix: only normalize the prefix separator, not underscores within the tool name suffix.

2. **All scripts missing auth headers** — The `callClaude()` function in each script needs `x-api-key` and `anthropic-version` headers added manually. Template:
   ```js
   headers: {
     "Content-Type": "application/json",
     "x-api-key": process.env.ANTHROPIC_API_KEY,
     "anthropic-version": "2023-06-01",
   },
   ```

3. **config.env format** — Keys use `KEY = 'value'` format with spaces and quotes. Scripts load via: `source <(sed "s/ *= */=/g; s/'//g" config.env)`

## Registry Format Reference

### TML Source
```
@tool.id
desc: What this tool does and when to use it
GET /endpoint
auth: none|bearer|apikey
in:
  param:type!          "description"
  param:type=default   "description"
out: {response_shape}
tags: keyword tags for search
---
```

### .min Index
```
#oatr1|API Name|https://base.url
tool.id|METHOD /path|param:t!,param:t=default|{output}|tags
```

Type codes: `s`=string, `n`=number, `i`=integer, `b`=boolean, `a`=array, `o`=object
`!`=required, `?`=optional, `=value`=default
