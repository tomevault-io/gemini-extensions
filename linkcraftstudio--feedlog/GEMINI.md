# AGENTS.md

This file provides guidance to coding agents (Claude Code, Cursor, Codex, Copilot, etc.) working with this repository. Human contributors: read [`CONTRIBUTING.md`](./CONTRIBUTING.md) first.

## Tech Stack

| Layer | Choice |
|-------|--------|
| Framework | **Nuxt 4** (Vue 3 + Nitro engine) |
| UI | **shadcn-vue** (new-york style) + **Tailwind CSS v4** + **Lucide icons** |
| State | **Pinia** |
| SEO | **@nuxtjs/seo** |
| Auth | **better-auth** (email/password + Google OAuth + GitHub OAuth + admin plugin) |
| Database | **PostgreSQL** (Drizzle ORM, postgres-js driver) |
| Deployment | CF Workers / Vercel / Docker (Nitro preset switching) |
| Package Manager | **pnpm** |

## Development

```bash
pnpm install                    # Install deps (postinstall runs nuxt prepare)
pnpm dev                        # Dev server at http://localhost:3000
pnpm build                      # Production build
pnpm preview                    # Preview production build

# Database (requires DATABASE_URL env var)
pnpm generate:migration         # Generate Drizzle migration after schema changes
pnpm migrate                    # Run database migrations

# Cloudflare Workers deployment
pnpm build:cf                   # Build with NITRO_PRESET=cloudflare-module
pnpm deploy:cf                  # Deploy .output to CF Workers via wrangler

# UI components
pnpm dlx shadcn-vue@latest add <component>  # Add shadcn-vue component (run pnpm install first to ensure .nuxt exists)
```

## Architecture

### Frontend (`app/`)

- `pages/` тАФ File-based routing: home, login, post detail (`p/[slug]`), roadmap, admin dashboard (`dashboard/*`)
- `layouts/` тАФ `default` (public pages) and `dashboard` (admin)
- `components/ui/` тАФ shadcn-vue base components (no prefix, `pathPrefix: false`)
- `composables/` тАФ `useApiFetch` (request wrapper), `useAuth` (auth state), `usePermission` (permissions)
- `stores/` тАФ Pinia stores: `board`, `postDetail`
- `lib/md-editor-helper.ts` тАФ Workaround for md-editor-v3 nested in shadcn Dialog (prevents internal popups from closing the outer Dialog); read this first when hitting similar issues

### Backend (`server/`)

API routes follow Nuxt file-based routing (`server/api/`), organized by resource:

- `auth/[...all].ts` тАФ better-auth catch-all handler
- `boards/` тАФ Public board list
- `posts/` тАФ Post CRUD + voting
- `posts/[postId]/comments/` тАФ Comment CRUD
- `comments/[id]/` тАФ Comment edit, delete, like
- `admin/` тАФ Admin APIs (boards CRUD + reorder, posts management), requires admin role
- `seed.post.ts` тАФ Seed data endpoint

Server utilities (`server/utils/`, auto-imported by Nitro):
- `auth.ts` тАФ `getUserSession()`, `requireAuth()`, `requireAdmin()` (three-level auth helpers)
- `db.ts` тАФ `useDB()` returns Drizzle instance
- `slug.ts` тАФ Slug generation

### Database

- **Connection**: `server/db/index.ts`, using `postgres` driver + `drizzle-orm/postgres-js`
- **Schema**: `server/db/schemas/index.ts` as entry point, defines board, post, comment, commentLike, vote tables and all relations
- **Auth Schema**: `server/db/schemas/auth.ts` (generated by better-auth) + `auth-schema.ts` (root-level reference copy), defines user, session, account, verification tables
- **Migrations**: `server/db/migrations/`, managed via `drizzle-kit generate` and `drizzle-kit migrate`
- **Drizzle Config**: `drizzle.config.ts`, schema path is `./server/db/schemas/index.ts`
- **ID Strategy**: Business tables use UUIDv7 (time-ordered), auth tables use text IDs
- **v0.2 additions**: `pgvector` extension + similar-merge tables (`post_embedding`, `duplicate_group`, `duplicate_member`, `duplicate_suggestion`) in `server/db/schemas/index.ts`; embeddings use OpenAI `text-embedding-3-large` at 768d

### Auth

- **Server**: `server/utils/better-auth.ts` тАФ better-auth instance with Drizzle adapter + email/password + Google OAuth + GitHub OAuth + admin plugin
- **Client**: `app/lib/auth-client.ts` тАФ `createAuthClient()` with `adminClient()` plugin
- **API Route**: `server/api/auth/[...all].ts` тАФ catch-all route proxying to better-auth

### Shared Code (`shared/`)

Nuxt 4 `shared/` directory (importable from both frontend and backend):
- `schemas/` тАФ Zod validation schemas (board, post)
- `types/` тАФ TypeScript type definitions (board, comment, pagination, post)

## Code Conventions

- TypeScript throughout
- Add shadcn-vue components via CLI, not manually
- Run `pnpm install` before adding components (`.nuxt` directory must exist for path alias resolution)
- `~~/` alias points to project root (for server-side imports like `~~/server/db`, `~~/lib/auth`)
- `@/` or `~/` alias points to `app/` directory (for frontend imports)
