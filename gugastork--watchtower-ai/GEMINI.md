# CLAUDE.md

Instructions for AI assistants working on this repository.

## About This Repository

This is the knowledge layer of the WatchTower builders community. It contains prompts, workflows, tools, templates, case studies, summaries and curated resources about practical AI applications.

## Content Standards

- All content must be tested and real-world — no theoretical or untested material
- Every file must follow the content template from `templates/content-templates/`
- Contributions must include context: what it does, when to use it, limitations
- No AI-generated content that has not been reviewed and validated by a human

## Repository Structure

```
prompts/         → tested prompts (subdirs: claude-code, agents, research, marketing)
workflows/       → AI workflows (subdirs: claude-code, cursor, n8n, automations)
tools/           → plugins, CLI tools, open source utilities
templates/       → reusable project templates + content templates
case-studies/    → real experiments with results
summaries/       → distilled insights from papers, tutorials, discussions
resources/       → curated books, repos, articles
```

## Writing Style

- Use English for all content
- Be direct and practical — no filler text
- Use Markdown for all files
- File names: lowercase with hyphens (e.g. `code-review-with-context.md`)

## When Helping Contributors

- Point them to the correct content template in `templates/content-templates/`
- Remind them to check CONTRIBUTING.md for quality standards
- Suggest the correct directory and subdirectory for their content
- Help them structure content clearly but do not generate fake examples or untested content
