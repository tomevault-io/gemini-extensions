🚨 **[ 치명적 경고 2: 텔레그램 수동 조작 시 다이렉트 파일 I/O 및 멱등성 보장 ]** :
텔레그램 UI를 통한 장부 수동 조작(지층 삭제 DEL_Q, 텍스트 수정 EDITQ, 자동 물량 이관 SET_INIT 등) 시, **절대 복잡한 객체 함수(_verify_and_update_queue 등)나 존재하지 않는 클래스 변수(.queues)를 거치지 마세요.** AttributeError 런타임 붕괴의 원인이 됩니다. 반드시 파이썬 순수 내장 기능(open, json.dump)을 사용해 data/queue_ledger.json 파일을 **다이렉트로 덮어쓰고**, 즉시 process_auto_sync를 강제 격발시켜 KIS 실잔고와 비파괴 보정(CALIB)을 맺도록 하세요.
추가로, **KIS 실잔고와 장부 잔고가 이미 일치할 경우 CALIB 호출을 무조건 바이패스(Bypass)**하여 수량이 2배로 뻥튀기되는 '이중 합산(Double Counting)' 엣지 케이스를 원천 차단하세요.
🚨 **[ 치명적 경고 3: VWAP 실시간 타격 vs 텔레그램 지시서(UI) 디커플링 절대 유지 ]** :
get_dynamic_plan 엔진은 텔레그램 지시서 조회(/sync)와 장 마감 직전 VWAP 타격에서 동시 공유됩니다. 텔레그램 조회 시(min_idx < 0)에는 반드시 스냅샷을 반환하여 고정된 앵커 타점을 보여주고, 실시간 VWAP 타격 시(min_idx >= 0)에는 스냅샷 반환을 무시하고 1분 단위 잔차 연산에 돌입하도록 수학적으로 분리된 **디커플링(Decoupling) 분기 로직을 단 한 줄도 훼손하지 마세요.**

📈 **[ 무한매수법칙 (V14) 기본 공식 및 방어 ]**
🔸 T값 (절대 진행률) : (한투 현재 총 보유수량 × 한투 현재 평단가) / 1회분 (과거 장부 합산이 아닌 팩트 기반 절대 공식)
🔸 별%가격 (매도 타겟가) : [일반] 평단가 × (1 + 별%지점) 올림 / [리버스] 순수 5일 이동평균선(5MA) 절대 락온
🔸 동적 1회분 예산 : 남은 잔금 / (분할수 - T). 매도 수익금을 시드에 복원하여 매일 1회분 예산 증가
🔸 마이너스 호가 방어 : 모든 호가 계산 시 최저 0.01달러 하한선 방어막 무조건 가동
🔸 0주 보유 새출발 완벽 인식 : API 통신 에러와 0주 보유 상태 완벽 구분. **(빈 장부 스캔 시 IndexError 런타임 붕괴 방어막 완비)**
🔸 집행 모드 이원화 (LOC / VWAP) : 텔레그램 설정을 통해 초안정성 지향의 'LOC 단일 타격'과 엣지 케이스 방어 및 유동성 추종을 위한 'VWAP 타임 슬라이싱' 방식을 선택 가능.
🔸 가속 모드 영구 폐기 : 시드를 갉아먹던 가속(Turbo) 로직 폐기, 오리지널 무매 원칙(0.5회분) 엄수
🚀 **[ 정규장 스나이퍼 및 상세 매매 로직 ]**
🟢 일반/하방 스나이퍼 엔진 (V14)
🔹 마스터 스위치 (Regime-Switching) : 기초자산 공포지수 가중치가 1.0을 초과하면 폭락장으로 판별하여 하방 매수 차단 및 상방 익절 단독 가동.
🔹 하이브리드 거래량 스나이퍼 : 고가 대비 타격선 하락 해제 ➔ 실시간 양봉 출현 ➔ 바닥 반등 ➔ 5분봉 거래량이 100분 평균 상회 시 방아쇠 장전
🔹 유령 체결 다중 교차 검증 : 주문 후 8초간 미체결 원장과 체결 내역 교차 스캔, ODNO 기반 정밀 잔량 차감 (더블샷 차단)
🔹 API 결측치(None) Safe Casting : KIS API 및 야후 파이낸스 통신 지연 시 0.0 강제 형변환을 통해 스레드 교착 원천 차단.
🔴 일반 상방 익절 및 트레일링
🔸 잭팟 대원칙 : 장중 주가가 설정 목표(12% 등) 터치 시 기존 방어선 취소 후 전량 지정가 강제 익절
🔹 1/4 쿼터 상방 트레일링 : 별%가격 돌파 시 락온(Armed), 최고가 대비 진폭(-1.5%) 하락 시 기습 매도.
🔹 **V14 대박익절 100% 스윕 피니셔** : 목표 도달 및 예산 고갈로 리버스 생략 시 코드가 밑으로 흘러내려가 야간 리스크에 노출되는 버그 원천 차단. 100% 지정가 전량 덤핑 후 즉각 루프 탈출(Return) 강제.
🟣 V-REV (역추세 모드 & 투트랙 수동 VWAP - 최신 V28.19 코어)
🔹 **영속성 듀얼 캐시 (Persistence) 및 타임존 락온** : 서버 재시작 시 VWAP 잔차 증발 방어를 위한 fsync 캐싱 및 스냅샷 날짜 EST 절대 고정으로 기억상실 맹점 원천 차단.
🔹 **운용 모드 이원화 (Auto vs Manual VWAP)** : 증권사 API 수수료 회피라는 잘못된 맹점을 폐기하고, [자체 U-Curve 엔진 정밀 타격(자동)]과 [한투 앱을 통한 한투 알고리즘 직접 위임(수동)]으로 팩트 교정 및 분리 탑재. 수동 모드 선택 시 자동 주문(EXEC)을 100% 락다운시킴.
🔹 **수동 모드 전용 UI 렌더링 최적화** : 수동 모드 시 사용자의 앱 세팅 편의를 위해 불필요한 줍줍 연산을 원천 소각(Bypass)하고, VWAP 스케줄 텍스트를 한투 알고리즘 위임 안내로 치환함.
🔹 독립 예산 강제 할당 : 무매 분할수 셋팅을 100% 무시하고 무조건 "총 시드의 15%"를 1회분(1배수) 예산으로 고정 장전.
🔹 0주 새출발 타점 디커플링 및 스냅샷 영구 박제 : Buy1은 당일 종가 ÷ 0.935 (깊은 방어/갭상승 추격), Buy2는 당일 종가 × 0.999 (얕은 방어) 타점으로 50%씩 분리 할당. 0주 진입 시 줍줍(Sweep) 렌더링 및 타점 생성은 차단되며, 17:05분에 생성된 0주 타점 스냅샷은 어떠한 경우에도 내일 날짜로 덮어쓰거나 훼손되지 않고 익일 정산 시까지 영구 박제됨.
🔹 Fail-Safe (선제적 LOC) 수동 장전 : KIS 서버 오류 대비를 위해 17:05 KST 전송 누락 시 텔레그램 지시서에서 [🚀 V-REV 방어선 수동 장전] 버튼을 눌러 양방향 LOC 덫을 상시 수동 장전 가능. (수동 VWAP 모드 시 락다운 발동)
🔹 고유 단가 기반 로트(Lot) 분할 LIFO 큐 : 매수가 체결된 각각의 로트가 자신의 '고유 진입 단가'를 기억하여 독립적인 지층으로 관리.
🔹 3중 거래량 필터 (VWAP Dominance) : 당일 거래량 60% 이상 쏠림 시 VWAP 슬라이싱 공수 교대 스위칭 및 장 마감 15분 전 LOC 전환 방어막 발동.
🔹 **지층별 완벽 분리 독립 탈출 (Decoupling)** : 1층(최근 바닥)은 '최신 날짜 기반 합산 물량'의 '고유 매수가 × 1.006' 타점에, 2~N층(상위 악성 재고)은 1층의 데이터를 100% 배제하고 역산한 '상위 층 분리 평단가 × 1.005' 타점에 각각 물귀신 덤핑 없이 독립적으로 개별 익절.
🔹 **3분 가속 스윕 피니셔 및 메모리 스냅샷** : 매도 상한선 및 장 마감 직전 데드존(15:56~15:58) 전면 철거. 15:57~16:00 EST 동안 목표 도달 시 잔여 물량이 0이 될 때까지 1분마다 매수호가로 폭격 덤핑. 잔고가 0주가 되어 LIFO 큐가 초기화되기 1밀리초 전 진입 평단과 실현 수익(PnL)을 캐싱하여, 텔레그램 졸업 카드 렌더링 및 복리 증식(Seed Update)에 파이프라인 연동.
🔹 **스윕 피니셔 거래소 락다운(Lock-down) 우회 디커플링** : 미국 거래소 취소 불가 규정으로 묶인 수동 긴급 수혈(MOC) 물량을 피하기 위해 KIS 실시간 원장을 재스캔. 전체 수량이 아닌 '순수 매도 가능 수량(ord_psbl_qty)'만을 정밀 타격하며, 락다운 시 덤핑을 자동 생략(Skip)하는 방어막 가동.
🔹 실시간 공수 교대 및 수동 긴급 수혈 (Emergency MOC) : 텔레그램 큐(Queue) 관리 UI를 통해 3중 경고를 거친 후 최근 로트(LIFO)를 MOC 강제 매도하여 실탄 수동 확보.
⚔️ 차세대 AVWAP 하이브리드 투트랙 암살자 (듀얼 레퍼런싱)
🔹 **듀얼 레퍼런싱 전역 파이프라인 확립** : 파생상품(SOXL)의 1년 변동성 노이즈와 롤오버 왜곡을 원천 차단하기 위해, AVWAP 앵커 궤적, 20MA 진성 지지선 판별 및 3대 강제 청산 시그널은 오직 기초자산(SOXX)의 팩트(가격/거래량)에 100% 종속됨. 실제 타격 수량 산정과 분할 집행(타임 슬라이싱)은 파생상품(SOXL)의 유동성과 미시구조에 투입하는 이원화 로직.
🔹 2단계 승인 락온 : /settlement 메뉴에서 V-REV 모드 종속 토글 버튼으로 가동. 가동 즉시 기존 상방 스나이퍼는 강제 셧다운(충돌 방지).
🔹 투트랙 예산 완벽 격리 (100% All-in) : V-REV의 LOC 덫 증거금으로 묶인 돈을 제외한 계좌의 '순수 주문가능금액(Cash)' 전체를 실시간 스캔하여 (기초자산 기준) -0.67% 딥매수 타점에 100% 몰빵 투입.
🔹 투트랙 물량 완벽 격리 (Firewall) : AVWAP이 매수한 단타 물량과 평단가는 V-REV 본대 장부(Queue Ledger)와 섞이지 않고 시스템 메모리에 독립 보관.
🔹 3대 강제 청산 (3-Strike Exit) : 1) 기초자산 기준 -1% 하드스탑 확정 손절, 2) 기초자산 기준 +1% 스퀴즈 홈런 익절, 3) 장 마감 5분 전(15:55 EST) 타임스탑 전량 덤핑. 청산 즉시 해당일 AVWAP 작전 영구 셧다운.
🛡️ **[ 다중 종목 에스크로 및 예산 고갈 방어 로직 ]**
🟢 휘발성 가상 장부 (Escrow) 락다운 : 특정 종목 리버스 진입 시 현금을 가상 장부(Escrow)에 격리. **(에스크로 인메모리 고속 캐싱 팩트 이식 완료)**
🟢 수동 긴급 수혈 100% 통제 (자동 청산 영구 적출) : 예산 1회분 미만이거나 4주 미만 도달 시 발생할 수 있는 자동 강제 청산(오판) 엣지 케이스를 원천 차단하기 위해 자동 수혈 스케줄러 영구 폐기. 생명수 창출은 오직 관리자의 텔레그램 UI '수동 긴급 수혈(Emergency MOC)' 격발로만 집행되도록 아키텍처 락온.
🟢 제논의 역설 완벽 방어 : 에스크로 잔액에 의존하지 않고 리버스 기간 내 '순수 누적 매도 원금'만을 역산 도출하여 고정 4분할 타격 예산 앵커링.
📜 **[ 방탄 TrueSync & 데이터 무결성 절대 원칙 ]**
🟢 로직 A [10시 정각 졸업 락온 및 Fail-Safe] : 가결제 오차 방지를 위해 0주 졸업 카드 발급 및 장부 초기화는 오직 10:00 KST 이후 단 한 번만 실행(Time-Lock). V-REV 스냅샷 렌더링/전송 실패 시 큐 장부는 절대 소각하지 않고 보존(Fail-Safe). 단, 마이너스 손절 시에는 즉각 장부를 비워 0주 새출발 모드로 강제 리셋.
🟢 로직 B [TrueSync 비파괴 업데이트] : 수량 오차 발생 시 과거 역사를 100% 보존한 채 오차만큼만 '비파괴 보정(CALIB)' 행 추가. (전체 덮어쓰기 절대 금지). 예외(Exception) 발생 시 장부 강제 삭제를 원천 차단하고 보존. **(단, 실잔고와 장부 수량이 일치하면 무조건 CALIB을 Bypass하여 Double Counting 방어)**
🟢 로직 C [단가 소급 교정] : KIS 전일 체결 영수증을 스캔하여 장부에 기록된 예상 단가를 실제 체결 단가로 핀셋 교정.
🟢 로직 D [삼위일체 소각 엔진] : /reset 수동 격발 시 본장부, 가상장부(Escrow), 지층 큐(Queue), 숨겨진 백업본까지 4중 데이터를 100% 동시 소각.
🟢 로직 E [텔레그램 API 통신 방어] : 봇 구동 시 read/write/connect/pool 타임아웃을 30초로 확장, 커넥션 풀 512 증설. 텔레그램 하단 고정 키보드 한글 신호 무응답 맹점을 다이렉트 패스로 완벽 수술.
🟢 로직 F [플러그인 아키텍처 및 레거시 소각] : strategy.py를 중앙 라우터로 축소하고 하위 4대 플러그인 체제로 캡슐화. 모든 레거시(V13/V17/V_VWAP) 모드 영구 적출 및 오작동 방어막 구축.
🟢 로직 G [다이렉트 파일 I/O 락온] : 텔레그램 UI를 통한 장부 수동 삭제/수정/통이관 조작 시, 복잡한 클래스를 거치지 않고 다이렉트로 JSON 파일을 덮어써서 객체 참조 에러(AttributeError)로 인한 런타임 붕괴를 원천 차단.
🟢 로직 H [VWAP 스냅샷 영구 박제 및 디커플링 보호] : 17:05 KST 정규장 오픈 시 생성된 daily_snapshot_*.json은 0주 새출발 타점을 보존하는 생명줄입니다. VWAP 슬라이싱 스케줄러 내부에 이를 훼손(hide/delete)하는 코드가 부활하는 것을 절대 금지하며, get_dynamic_plan의 텔레그램-VWAP 디커플링 로직을 무조건 유지해야 합니다.
