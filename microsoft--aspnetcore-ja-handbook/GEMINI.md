## Code Review

### 日本語表記

- 日本語の誤字・脱字を指摘してください。
- カタカナ表記ゆれを指摘してください（例：「メモリ」と「メモリー」）。
- 長音符号は、技術文書の表記統一として一般的なルールに従い、3音以上（3モーラ以上）の語尾の長音は原則省いてください。
  - 「サーバー」「ユーザー」などの2文字の場合は原則付与します。
  - 「モニタ」「コンピュータ」など3文字の場合は原則付与しませんが、「コントローラー」のように一般的に長音符号を使用する表記は許容されます。

### 技術的正確性

- ASP.NET Core / .NET の API 名やクラス名が正しいか確認してください。
- コードサンプルのシンタックスが正しいか確認してください。
- 公式ドキュメントの URL が正しい形式か確認してください。
- バージョン情報（.NET 10, .NET 9 など）が文脈と整合しているか確認してください。

### マークダウン構文

- リンクの記法 `[テキスト](URL)` が正しいか確認してください。
- コードブロックの言語指定（csharp, xml など）が適切か確認してください。
- テーブルの列数が揃っているか確認してください。

### 対象読者への配慮

- 対象読者（C# 以外の言語経験者）にとって丁寧な表現かつ内容になっているか確認してください。
- 他言語（Java, Python, TypeScript, Go）との比較説明が正確か確認してください。
