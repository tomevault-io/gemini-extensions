# Project language

**All project-facing content must be in English**, regardless of the language used to chat with the AI.

This applies to:

- **Code**: variable names, function names, types, file names (except product/brand terms).
- **Comments and JSDoc**: explanations in code and documentation blocks.
- **Commit messages**: write them in English.
- **Docs in the repo**: README, CONTRIBUTING, rule files (e.g. `.cursor/rules/*.mdc`), and any other project documentation.
- **UI copy**: if the app has user-facing text, keep it in English unless the product explicitly targets another locale.

You may still communicate with the AI in any language; the rule only affects what gets written into the codebase and project assets.
