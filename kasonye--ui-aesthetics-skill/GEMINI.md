# UI Aesthetics

Apply this rule when the task is about improving how a web UI looks or feels.

## Core expectations

- preserve the requested scope and information density
- fix hierarchy, spacing, and typography before adding polish
- keep colors restrained and role-driven
- keep default states quiet and interaction states distinct
- use depth and motion only when they clarify structure or interaction
