# catbird-mls

## Overview

`catbird-mls` is the core MLS (Messaging Layer Security) SDK for all Catbird clients. It implements OpenMLS 0.8 group cryptography, a platform-agnostic orchestrator (sync, recovery, messaging policy), and UniFFI bindings so Swift/Kotlin call into a single Rust library. It also contains voice encoding (Opus/OGG), blob crypto (AES-256-GCM), and a WASM client facade — making it a monolith that should eventually be split. The spec that governs all behavior is `docs/MLS_CLIENT_PROTOCOL.md`.

## Architecture

### The Four Platform-Agnostic Traits

```
MLSOrchestrator<S, A, C, M>
  S: MLSStorageBackend     — persistence (conversations, messages, groups)
  A: MLSAPIClient          — HTTP to mls-ds and nest
  C: CredentialStore       — signing key, DID, device UUID (keychain/keystore)
  M: MlsCryptoContext      — synchronous OpenMLS operations
```

On iOS/Android: each trait is a UniFFI callback interface (Swift/Kotlin provides the impl).
On WASM/CLI: Rust implementations used directly.

### Async Runtime Bridging

UniFFI callbacks are synchronous. The orchestrator is async. Bridge: `async_runtime::block_on` (`src/lib.rs`), which uses `tokio::task::block_in_place` inside Tokio or falls back to the crate-internal 4-thread runtime. **Keep callbacks fast — they block a thread.**

### Recovery State Machine

This crate exposes a **5-state** `ConversationState` enum
(`src/orchestrator/types.rs:154`) tracking **per-conversation, orchestrator-
internal** lifecycle:

```
Initializing → Active → ForkDetected → NeedsRejoin → Failed
```

- `Initializing` — group is being created but not yet committed
- `Active` — healthy; sending and receiving messages
- `ForkDetected` — fork detected during decryption; fork-readd in flight
- `NeedsRejoin` — flagged for deferred External Commit rejoin
- `Failed` — terminal local failure

It's stored in `MLSOrchestrator.conversation_states`
(`Mutex<HashMap<ConversationId, ConversationState>>`) and read/written by
`messaging.rs`, `recovery.rs`, and `groups.rs`.

### Layering: Rust 5 states vs spec 7 states

**The `ConversationState` enum here is NOT the same state machine as the
7-state conversation recovery machine in `docs/MLS_CLIENT_PROTOCOL.md §8.1`.**

The spec's 7-state machine (`HEALTHY`, `EPOCH_BEHIND`, `GROUP_MISSING`,
`NEEDS_REJOIN`, `RECOVERING`, `UNRECOVERABLE_LOCAL`, `RESET_PENDING`) is a
**client-layer, cross-platform contract** that lives at the Swift / Kotlin /
catmos / catmos-cli layer. Each platform implements it independently:

- **iOS**: `CatbirdMLSCore/Sources/CatbirdMLSCore/Models/ConversationRecoveryState.swift`
  (Bravo, Cluster B). Derives persistent states from `MLSConversationModel`
  booleans (`needsRejoin`, `needsReset`, `isUnrecoverable`); transient
  states live in `MLSRecoveryManager.transientStates`. iOS does **not**
  call into `MLSOrchestrator` at all — it uses the low-level `MlsContext`
  UniFFI object directly and runs its own sync + recovery loop in Swift
  (`MLSConversationManager+Sync.swift`).
- **catmos / catmos-cli / WASM clients**: use `MLSOrchestrator` via
  `CatbirdClient` / `WasmClient`. The orchestrator's 5-state
  `ConversationState` is a smaller internal bookkeeping enum; the spec's
  7 states are currently mapped loosely onto it. Concretely:
  - `Initializing` has no spec equivalent (pre-creation).
  - `Active` ≈ `HEALTHY`.
  - `ForkDetected` is a **Rust-specific** fork-readd path for
    `recover_fork_by_readding`; the spec treats forks as a flavor of
    `NEEDS_REJOIN`.
  - `NeedsRejoin` ≈ `NEEDS_REJOIN`.
  - `Failed` ≈ `UNRECOVERABLE_LOCAL`.
  - `EPOCH_BEHIND`, `GROUP_MISSING`, `RECOVERING`, and `RESET_PENDING`
    have **no direct Rust equivalent** — they're observed indirectly via
    counters in `RecoveryTracker` (in-flight rejoin = `RECOVERING`;
    group-not-found propagates from `MlsContext::get_epoch` failures;
    server reset is observed via `GroupResetEvent` handled entirely at
    the client layer).

**Why two machines?** The Rust orchestrator's state tracks the attempt
lifecycle inside its own sync loop: "am I currently allowed to try a
rejoin?" It needs a small vocabulary (5 states) because its only
side-effects are "try to send an External Commit" or "drop the message."
The platform-layer state tracks what to show the user, what persists on
disk, and when to escalate to server-mediated recovery (`S1.1`
pyramid) — concerns that don't exist in the Rust core.

**Do not conflate them in code reviews.** If a PR talks about "the 7
states," it's the client-layer machine. If it talks about `ConversationState`
or `ForkDetected`, it's the orchestrator.

**Drift to watch:** when the spec or client implementation diverges from
what `messaging.rs` writes into `conversation_states`, check whether the
client/server mapping is still semantically correct. The `RecoveryTracker`
counters (`failed_rejoins`, `last_global_rejoin_at`) are the actual
enforcement mechanism for the S1 rate-limits; the `ConversationState`
enum is just diagnostic bookkeeping.

Key `RecoveryTracker` properties:
- **Global 30s minimum** between any External Commit across all conversations
- **Per-conversation backoff**: 30s → 2m → 10m, max 3 attempts
- **GroupInfo 404 circuit breaker**: 3 consecutive 404s suspends attempts
- **Sequencer failover**: 3 failures over 2min triggers `request_failover`
- Trackers are **in-memory only** (reset on restart = natural cooldown)

### Identity Model

- `conversationId` = stable, server-assigned. Use for API calls.
- `groupId` = current MLS group hex ID. Changes on reset. Use for crypto calls.
- After reset: `ConversationView.group_id != ConversationView.conversation_id`. Never confuse them.

### Wire Format

```json
{"version":1,"messageType":"text","text":"..."}
```
camelCase fields (matches iOS Codable). Fallback: raw bytes as UTF-8 (legacy). Embeds carry encrypted blob references, not raw data.

### SQLCipher Configuration

32-byte plaintext header (iOS WAL compat), 256K KDF iterations, 4096 page size, deterministic salt = `SHA-256(key)[..16]` hex. Must match exactly across restarts or database is unreadable.

### Traffic-Analysis-Resistant Padding

Outgoing ciphertext padded to buckets `[512, 1024, 2048, 4096, 8192]` bytes using `[4-byte BE length][ciphertext][zeros]`.

## Coding Conventions

- `cargo fmt` + `cargo clippy --all-targets --all-features` must be clean before commit
- All spec constants in `orchestrator/constants.rs` — never hardcode timers/thresholds
- Orchestrator impl blocks split across files (`groups.rs`, `messaging.rs`, `sync.rs`). Keep files under ~200 lines.
- Trait default methods = no-ops for backward compat. New required methods must have defaults until all platforms implement them.
- FFI entry points use `std::panic::catch_unwind`. Never let panics cross the FFI boundary.
- Use `async_runtime::block_on`, not `RUNTIME.block_on()` directly (handles catmos/Tauri case).
- `MLSMessagePayload` JSON fields must match iOS Codable exactly (camelCase via `#[serde(rename_all = "camelCase")]`).

## Gotchas

- **Async bridge blocks threads.** UniFFI callbacks calling async orchestrator code do `block_in_place`. If Swift holds locks during a callback → deadlock.
- **GroupId changes on reset; ConversationId does not.** Always use `group_id` for crypto, `conversation_id` for API.
- **SQLCipher plaintext header = 32 bytes.** Non-standard config for iOS GRDB WAL compat. Changing the key or salt derivation makes old databases unreadable.
- **Own-commit dedup = SHA-256(ciphertext).** TTL 300s. If you change commit construction, verify the hash still matches echoed commits.
- **`groups_being_created` set prevents sync from deleting nascent groups.** Group is added before server call, removed after. Sync must check this set.
- **Voice encoding compiles on all native targets including WASM.** `audiopus_sys` builds statically even for WASM builds. The WASM facade doesn't expose voice, but the code links.
- **Android NDK linkers must be on PATH.** `.cargo/config.toml` specifies per-ABI linkers. `build-android.sh` auto-detects from `~/Library/Android/sdk/ndk`.
- **`uniffi-bindgen` must match the library version.** If you update UniFFI in Cargo.toml, rebuild bindgen before regenerating Swift/Kotlin.
