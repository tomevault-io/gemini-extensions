# AGENTS

## Design Context

### Users
这个项目的主要用户是中文开发者，通常在编码、排查问题、查命令、看配置项时打开这张速查页。
他们使用时的核心场景不是“沉浸式阅读”，而是“快速扫读、快速定位、快速确认”。
界面需要优先服务信息检索效率，让用户在高密度内容里迅速找到模块、分组、字段和说明。

### Brand Personality
品牌气质聚焦为：高效、锐利、现代。
页面给人的感受应当是专业、可信、干净利落，而不是热闹、营销化或装饰驱动。
情绪目标是让用户在短时间内获得“我能很快看懂”“我能马上找到答案”的信心。

### Aesthetic Direction
视觉方向应保持工具型信息界面的基因，优先强调结构层级、对比度、扫读节奏和模块识别度。
整体更接近开发者工具面板、技术文档速查页、终端辅助参考面板，而不是 SaaS 营销页、品牌展示页或插画型落地页。
需要同时支持浅色和深色两套主题，但两套主题都要保持高对比、强可读、色盲友好，并避免依赖颜色本身传达唯一语义。
动画应极少且克制，默认偏静态，只有在明显帮助理解或反馈时才允许出现，并应兼容 reduced motion。

### Design Principles
1. 扫读效率优先于装饰表现。
   任何视觉决策都应先服务于“更快定位信息、更少认知负担、更清晰层级”。

2. 高对比与可访问性是硬约束，不是锦上添花。
   所有主题都应满足 WCAG AA，兼顾色盲友好、减少动画和清晰焦点态。

3. 现代感来自秩序、比例和节奏，不来自花哨特效。
   可以锐利，可以有技术感，但不要依赖重阴影、夸张渐变、炫技动效或视觉噪声。

4. 展示层可以重做，但同步链路不能被破坏。
   上游抓取、解析、映射、生成的逻辑必须保持稳定；设计调整只发生在前端展示层与样式表达层。

5. 信息密度要高，但不能压垮阅读。
   允许紧凑布局，但必须通过分组、留白、字重、字号、颜色层级和模块边界维持清晰可读。

## Repository Hygiene

- 使用 AI 生成、但与项目源码、运行链路、部署链路无关的过程产物，默认不入库，必须加入 `.gitignore`。
- 当前按此规则忽略的内容包括：`.impeccable.md`、`.ace-tool/`、`docs/superpowers/`。
- 如果后续再产生新的本地 AI 计划、spec、过程记录或工具缓存，除非用户明确要求提交，否则默认先忽略。
