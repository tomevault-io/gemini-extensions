# Agent Instructions

## About opencv-ruby

Ruby bindings for OpenCV. The C++ bindings are auto-generated by [ruby-bindgen](../ruby-bindgen/) and then compiled as a Ruby C extension using [Rice](https://github.com/ruby-rice/rice).

## Key Paths

- `ext/` - Generated C++ binding source files (`*-rb.cpp`, `*-rb.hpp`, `*-rb.ipp`)
- `ext/rice-bindings.yaml` - ruby-bindgen configuration for generating Rice C++ bindings
- `ext/cmake-bindings.yaml` - ruby-bindgen configuration for generating CMake build files
- `ext/manual_updates.md` - **Critical:** Manual fixes that must be applied after every regeneration
- `ext/manual_updates.diff` - Reference diff of all manual changes
- `lib/` - Ruby source files
- `docs/` - API documentation

## Generating Bindings

Bindings are generated using ruby-bindgen from `/mnt/c/Source/ruby-bindgen`. The process has two stages:

```bash
cd /mnt/c/Source/ruby-bindgen

# 1. Generate Rice C++ source files
time bundle exec ruby -Ilib bin/ruby-bindgen /mnt/c/Source/opencv-ruby/ext/rice-bindings.yaml

# 2. Generate CMake build files (must run after Rice generation)
time bundle exec ruby -Ilib bin/ruby-bindgen /mnt/c/Source/opencv-ruby/ext/cmake-bindings.yaml
```

**IMPORTANT:** Always use `time` and ensure the timing output is visible to the user (not truncated). If output is long, capture timing separately, e.g. pipe main output to `tail` or redirect stdout so the `time` result is not lost.

### After Regeneration

After generating bindings, you **must** apply all manual updates documented in `ext/manual_updates.md`. These include:

- **Manual `#include` directives** — Required headers not automatically generated
- **DNN module namespace fixes** — Remove versioned inline namespace suffixes
- **Template builder modifications** — `if constexpr` guards for Matx/Vec constructors and methods
- **Class inheritance fixes** — Parent class specifications for proper Ruby class hierarchies
- **OpenCV version guards** — `#if RUBY_CV_VERSION >= XXX` guards for version-specific features
- **Miscellaneous fixes** — Default argument removals, type corrections, duplicate removals

After applying all manual updates, generate the diff file:

```bash
git diff -- ext/ ':(exclude)ext/manual_updates.md' > ext/manual_updates.diff
```

### Important Rules

- **NEVER use `git checkout` to restore files in ext/.** The generated branch contains freshly generated code. Checking out from main or other branches will overwrite it and require full regeneration.
- **Include order matters.** OpenCV/system headers go before the primary header; local project headers go after. Mark manual includes with `// Manual` comment.
- **Verify every change.** After editing a file, re-read it to confirm changes are present before moving on.

## Commits

## Building

The project uses CMake with presets for building the C++ extension. OpenCV is managed via vcpkg on Windows.

```bash
cd ext

# Configure
cmake --preset linux-debug    # or linux-release, msvc-debug, etc.

# Build
cmake --build --preset linux-debug
```

The built extension is placed in `lib/` (Linux/macOS) or `lib/<ruby-version>/` (Windows). See `docs/cmake-build.md` for full details.

## Testing

```bash
bundle exec rake test
```

## ruby-bindgen Configuration

See `ext/rice-bindings.yaml` for the full configuration including:

- `match` / `skip` — Which OpenCV headers to process or exclude
- `rename_methods` / `rename_types` — Ruby name overrides
- `skip_symbols` — Symbols to exclude (unexported, internal, broken)
- `clang-cl` / `clang` — Platform-specific compiler toolchain settings

For ruby-bindgen features and configuration docs, see `/mnt/c/Source/ruby-bindgen/docs/`.
