## Project Overview

Drapes UI is a library of animated canvas backgrounds for React/Next.js applications. The project consists of a collection of customizable, animated canvas components with a registry system for managing and displaying them with live previews and code generation.

## Development Commands

```bash
# Install dependencies
pnpm install

# Development workflow
pnpm dev                # Runs generate scripts + starts Next.js dev server with Turbo
pnpm run generate-dev       # Runs both generate-index and generate-codes
pnpm run generate-index # Regenerates backgrounds/index.ts from registry.config.ts
pnpm run generate-codes # Generates code.ts files for all backgrounds

# Build and lint
pnpm build              # Runs generate scripts + creates production build
pnpm start              # Starts production server
pnpm lint               # Runs Biome checks
pnpm format             # Formats code with Biome
```

## Architecture

### Background Registry System

The registry system is the core of Drapes UI, dynamically loading and managing background components:

1. **Registry Configuration** (`backgrounds/registry.config.ts`):
   - Central source of truth for which backgrounds are active
   - Controls ordering and "new" badge display
   - Backgrounds are registered via `RegistryEntry[]` with `folder` and optional `isNew` flag

2. **Auto-Generated Index** (`backgrounds/index.ts`):
   - **NEVER edit this file manually** - it's auto-generated by `generate-index.ts`
   - Automatically creates imports and registry calls based on `registry.config.ts`
   - Converts kebab-case folder names to PascalCase component names
   - Validates that both `index.tsx` and `config.ts` exist before including a background

3. **Registry Class** (`lib/registry.ts`):
   - Singleton pattern managing the `BackgroundRegistry`
   - Stores `BackgroundEntry` objects (config + component + isNew flag) in a Map
   - IDs are auto-assigned sequentially (1, 2, 3, etc.) based on registry order

### Background Structure

Each background lives in `backgrounds/<folder-name>/` with this structure:

```
backgrounds/<folder-name>/
  ├── index.tsx      # React component (default export)
  ├── config.ts      # Configuration (default export as Omit<BackgroundConfig, 'id'>)
  └── code.ts        # Auto-generated by generate-codes script
```

**Naming Convention**: Folder name MUST be kebab-case matching the component identity (e.g., `dot-grid` for `DotGrid` component).

### Background Config Schema

Every `config.ts` exports a configuration object with:

- `name`: Display name
- `description`: Brief description for UI
- `author`: Object with `name`, `imageUrl`, `redirectUrl`
- `tags`: Array of categorization strings
- `defaultProps`: Default values for all component props
- `controls`: Array of control definitions for the UI panel (slider, color, select, toggle, media-input)
- `code`: Object with 6 code snippet variations (usage/rawUsage, tsx/rawtsx, jsx/rawjsx)

The `id` field is omitted in config.ts and auto-assigned by the registry based on order.

### Code Generation

Two automatic code generation scripts maintain consistency:

1. **`generate-index.ts`**: Regenerates `backgrounds/index.ts` from `registry.config.ts`
2. **`generate-code.ts`**: For each background, generates `code.ts` with:
   - TSX → JSX conversion using Sucrase
   - Syntax-highlighted HTML using Shiki (vesper theme)
   - Usage examples based on defaultProps
   - Both raw code strings and HTML-rendered versions

These scripts run automatically on `pnpm dev` and `pnpm build`.

### App Routes

- `/` - Main gallery/homepage displaying all backgrounds
- `/bg` - Background preview route for individual backgrounds
- `/test` - Development testing route for new backgrounds (not in production)

## Adding New Backgrounds

1. **Test First**: Build the component in `app/test/page.tsx` to verify functionality
2. **Create Folder**: Add `backgrounds/<name>/` with `index.tsx` and `config.ts`
3. **Register**: Add entry to `backgrounds/registry.config.ts` (order matters)
4. **Run Dev**: Execute `pnpm dev` - generation scripts handle the rest

DO NOT manually edit `backgrounds/index.ts` or any `code.ts` files - they are auto-generated.

## Tech Stack

- **Framework**: Next.js 16 with App Router, React 19
- **Styling**: Tailwind CSS 4
- **Linting/Formatting**: Biome (not ESLint/Prettier)
- **Animation**: Canvas API, WebGL shaders, Three.js
- **Package Manager**: pnpm (required)

## Important Notes

- All background components must use `'use client'` directive (Canvas requires client-side rendering)
- Performance is critical - backgrounds use requestAnimationFrame, canvas optimization, and DPR scaling
- The registry order in `registry.config.ts` determines the ID assignment and display order
- When modifying backgrounds, always regenerate code with `pnpm run generate-codes` before committing
