# Userpoll — CBB Poll Voter Visualizer

## What this project is

A scraper + static site generator for college basketball poll data from `cbbpoll.net`. `userpoll.py` scrapes voter ballots, downloads team logos, and generates `index.html` and `ap-ballots.html` (inlining `style.css` and `script.js` at build time) for interactive visualization. Deployed at **https://les-champs.github.io/cbb-ballots/**.

## Repository structure

```
userpoll.py          # Main script: scraper, HTML generator (reads style.css + script.js at build time)
style.css            # All CSS — edited directly, inlined into both index.html and ap-ballots.html by userpoll.py
script.js            # All client-side JS — edited directly, inlined into both index.html and ap-ballots.html by userpoll.py
poll_simulator.py    # Standalone simulator to test/tune the Précision scoring formula
index.html           # Generated output — the live site (DO NOT hand-edit; regenerated by userpoll.py)
ap-ballots.html      # Generated AP ballots page (DO NOT hand-edit; regenerated by userpoll.py --rebuild)
data-YEAR.html       # Userpoll year fragment files (e.g. data-2025.html) — generated, DO NOT hand-edit
data-ap-YEAR.html    # AP25 year fragment files for index.html (e.g. data-ap-2026.html) — generated, DO NOT hand-edit
data-appage-YEAR.html # AP25 year fragment files for ap-ballots.html (clean IDs) — generated, DO NOT hand-edit
ballots.json         # Browser-exported poll data (too large for Claude to read — do not attempt)
cbb-logos/           # ~200 team logos in .webp format, used by index.html
thumbnail.png        # OG/Twitter card thumbnail
cbbpoll-YEAR.json    # Cached scrape data from cbbpoll.com (one per season, generated)
ap25-YEAR.json       # Cached scrape data from collegepolltracker.com (one per season, generated)
```

## How userpoll.py works

### CLI commands
```bash
python3 userpoll.py --week 18 --year 2026    # Scrape week and append to index.html
python3 userpoll.py --rebuild                 # Rewrite HTML (updated CSS/JS) preserving all week data
python3 userpoll.py --delete --week 18 --year 2026   # Remove a week
python3 userpoll.py --to-webp                 # Batch convert PNG logos to WebP
python3 userpoll.py --ap25 --year 2026        # Scrape AP25 data for a season to ap25-YEAR.json
python3 userpoll.py --ap25 --year 0           # Scrape all AP25 seasons
python3 userpoll.py --ap25-build --year 2026  # Build data-ap-YEAR.html + data-appage-YEAR.html, update index.html + ap-ballots.html
python3 userpoll.py --ap25-build --year 0     # Build all AP25 seasons
```

### Three layers in one file

1. **Scraping** (top of file): Fetches from cbbpoll.net via `urllib.request` + regex. Falls back through: homepage → seasons archive → `__NEXT_DATA__` JSON → user profile pages. Downloads logos as PNG, converts to WebP via PIL.

2. **HTML generation** (middle): Builds HTML tables — one `<tr>` per voter, 25 `<td>` cells for ranked teams. Three sections per week: Official, Provisional, Aggregate. Incremental: loads existing HTML (including all `data-YEAR.html` fragments), replaces/appends week blocks. `--rebuild` rewrites CSS/JS while preserving data (used after code changes). Only the most recent year's blocks are embedded inline in `index.html`; all prior years are written to `data-YEAR.html` fragment files. `--rebuild` also regenerates `ap-ballots.html`.

3. **Client-side JS** (`script.js`): All interactive features. This is the bulk of the complexity. Read from disk and inlined into `index.html` at build time.

### Key JS features
- **Week/Year navigation:** Picker UI with pinning (Alt+click locks week picker open), Shift+Arrow keyboard nav
- **Sort modes** (click column header): Alphabetic, Consensus (deviation from aggregate), Précision (weighted accuracy), Tenure (total ballot count)
- **Delta view** (click voter row): Rank changes vs previous week + weighted 5-week averages per team
- **Row controls:** 📌 pin to top across all weeks, 🖊️ highlight row, 📅 season history modal
- **Team highlighting:** Click any logo to glow-highlight all occurrences (5 color slots)
- **Team rank chart:** Shift+click any logo in a voter row to open an SVG line chart showing that team's rank in that voter's ballot across all weeks of the current season. NR is shown (dashed line) for weeks where the voter submitted a ballot but didn't rank the team; weeks with no ballot submitted are omitted. Uses the voter history modal.
- **5-Week Weighted row:** Injected in aggregate view, weighted avg ranks across all voters
- **Keyboard shortcuts:** Shift+A (aggregate), Shift+D (all deltas), Shift+J (Précision sort), Shift+P (save PNG), Shift+1-9 (delta lookback window), Shift+Arrows (navigate weeks/years)
- **Lazy loading:** Current week loads immediately; background loader processes remaining current-year weeks at 16-image concurrency (HTTP/2), then fetches all prior-year fragment files in parallel, then loads their images
- **AP Ballots toggle:** Clicking the "Official Ballots" / "Provisional Ballots" header cycles through Official → Provisional → AP Ballots → Official. In AP mode, the userpoll block is hidden and the corresponding AP25 week block is shown (loaded on demand via `loadApYear()`). Navigation (week/year pickers, keyboard) always operates on userpoll tabs; AP data rides along via the toggle state. On `ap-ballots.html`, the toggle is static ("AP Ballots") and non-interactive — AP weeks are the primary content.
- **AP mode stats routing:** When in AP Ballots mode on `index.html`, Shift+Y (avg rank table), Shift+G (rank spread chart), and Shift+U (season stats) all use AP block IDs and `getSortedApTabs()` so they read AP data. The rank spread chart and avg rank modal also auto-refresh on AP week/year navigation. `showAvgRankModal()` and `showSeasonStatsModal()` detect AP mode via `weekId.startsWith("week-ap-")` and `_apCurrentYear !== null` respectively.

### Précision scoring algorithm (EXPERIMENTAL — still being tuned)

The `computeVoterAvg()` JS function and `score_voters()` in `poll_simulator.py` implement the same formula:
- **Base distance:** Position-weighted distance of each ballot from `finalRanking` (5-week weighted avg of last 5 consensus rankings)
- **Position weight:** 1.0 at #1 → 0.5 at #25 (linear)
- **Week weight:** 0.5 at week 1 → 1.0 at final week (linear recency bias)
- **Boldness adjustment:** Reward for correctly deviating from weekly consensus toward the final ranking. Flat scalar (0.05), weighted by deviation magnitude, position-weighted
- **Miss penalty:** +1 per missed week on combined score; ≥5 misses or ≥3 consecutive → disqualified
- **Combined:** `avg - boldAdj + missedWeeks * penalty` (lower is better)

## poll_simulator.py

Mirrors the JS Précision equation in pure Python for testing. Features:
- `--from-json ballots.json` to score real ballot data (note: ballots.json is too large for Claude)
- `--sanity` / `--scenario` / `--sensitivity` / `--params` for demos and parameter tuning
- Parameterized via `PARAMS` dict — edit to test sensitivity
- When changing the Précision formula in JS, keep `poll_simulator.py` in sync

## AP25 scraper (collegepolltracker.com)

### Background

`collegepolltracker.com` hosts AP Top 25 individual voter ballots. Data is scraped separately from the userpoll data and displayed via the toggle cycle (Official → Provisional → AP Ballots). Seasons 2022–2026 are available.

### Site structure

- Grid page: `https://collegepolltracker.com/basketball/grid/{site_year}/{week_slug}`
  - `site_year` is one less than our year (e.g. 2026 season → site year 2025)
  - Week slugs: `pre-season`, `week-2` through `week-19`, `final-rankings` (stored as week 20)
  - Week 1 doesn't exist; invalid weeks redirect to homepage
- Voter page: `https://collegepolltracker.com/basketball/pollster/{slug}/{site_year}/{week_slug}`

### Available seasons

| Our year | Site year | Notes                        |
|----------|-----------|------------------------------|
| 2022     | 2021      | Weeks 2–19, final-rankings   |
| 2023     | 2022      | Weeks 2–18, final-rankings   |
| 2024     | 2023      | Weeks 2–19, final-rankings   |
| 2025     | 2024      | Weeks 2–18, final-rankings   |
| 2026     | 2025      | Pre-season, weeks 2–19       |

### Storage and build

- Cached JSON: `ap25-YEAR.json` — one file per season, cache-aware (re-run skips already-fetched weeks)
- Fragment files for `index.html`: `data-ap-YEAR.html` — one per season, written in a single batch pass (not per-week). Uses `week-ap-YEAR-WEEKNUM` IDs and `data-poll="ap"`.
- Fragment files for `ap-ballots.html`: `data-appage-YEAR.html` — written alongside `data-ap-YEAR.html` by `ap25_build_season`. Uses clean `week-YEAR-WEEKNUM` IDs, no `data-poll` attribute.
- AP tabs live in `<!-- AP-TABS --> … <!-- /AP-TABS -->` in `index.html`; `AP_FRAGMENT_YEARS` JS constant lists which years need fetching
- `ap-ballots.html` uses `FRAGMENT_YEARS` (set to the appage fragment years) and `FRAG_PATH_PREFIX = 'data-appage-'` so `loadYear()` fetches the right fragment files
- **IMPORTANT:** `ap25_build_season` writes both fragment files and updates index.html and ap-ballots.html in one pass per year. The old per-week `save_ap_html` approach caused O(n²) memory explosion — do not reintroduce it.
- `--rebuild` preserves existing AP tabs and `AP_FRAGMENT_YEARS` via `get_ap_info()`, and also calls `rebuild_ap_html()`

### Team name mapping

- `AP25_NAME_OVERRIDES` dict handles edge cases (e.g. "Pittsburgh" → "Pitt", "McNeese State" → "McNeese")
- `_normalize_ap_team_name()` auto-normalizes to PascalCase then checks overrides

## cbbpoll.com legacy scraper

### Background

The site `cbbpoll.com` hosts older seasons (2015-16 through 2021-22) that predate `cbbpoll.net`. The data is structurally different and requires a separate scraping path. This is a different site from `cbbpoll.net` (the current/modern source).

### Site structure (cbbpoll.com)

**Three page levels:**
1. **Poll page** (`/poll/YEAR/WEEK`) — lists all voters with ballot links, split into Official Ballots / Provisional Ballots / Non-submitters
2. **Ballot page** (`/ballot/ID`) — individual voter's 25-team ranking
3. **Overview** (`/overview/`) — only works for 2022 season; other years return 200 but show the same 2022 data (not useful for discovery)

**Available seasons and weeks:**
| Year param | Season    | Weeks                |
|------------|-----------|----------------------|
| 2016       | 2015-16   | 2-19 (no preseason)  |
| 2017       | 2016-17   | 0, 3-19              |
| 2018       | 2017-18   | 0, 2-20              |
| 2019       | 2018-19   | 0, 2-21              |
| 2020       | 2019-20   | 0, 2-20              |
| 2021       | 2020-21   | 0, 2-18              |
| 2022       | 2021-22   | 0, 2-20              |

Week 1 never exists (always 302 redirect). Invalid weeks also 302 to homepage. Years before 2016 redirect (302) to homepage.

### HTML formats on cbbpoll.com

**Ballot page** (`/ballot/ID`) — team entries:
```html
<span class="team-name">
  <span class=logo30>
    <img src='/static/img/logos_30.png' class='logo30-ucla' alt="University of California, Los Angeles Logo">
  </span> UCLA
</span>
```
Key fields: **display name** after the `</span>` (e.g., "UCLA", "Colorado State", "St. Bonaventure") and **CSS sprite class** (e.g., `logo30-ucla`, `logo30-colost`).

**Poll page** (`/poll/YEAR/WEEK`) — voter entries:
```html
<span class='ballot-list-item'>
  <a href="/ballot/16037">
    <span class=logo23>
      <img src='/static/img/logos_23.png' class='logo23-uva' alt="University of Virginia Logo">
    </span>Phillyfan3
  </a>,
</span>
```
Sections are delimited by "Official Ballots", "Provisional Ballots", and "Voters who failed to submit a ballot".

### Team name mapping challenge

cbbpoll.com uses human-readable display names (e.g., "St. Bonaventure", "Colorado State", "NC State") while our logos use PascalCase filenames (e.g., `StBonaventure.webp`, `ColoradoState.webp`, `NCState.webp`).

**Mapping strategy:**
1. Auto-normalize: strip spaces, periods, apostrophes, hyphens → PascalCase
2. Manual override dict for edge cases (e.g., "Ole Miss" → `OleMiss`, "Miami (FL)" → `MiamiFL`)
3. Voter flair logos use alt text like "University of Virginia Logo" — map via full-university-name → logo filename dict

### Scraping approach

**Phase 1 — Poll pages:** Fetch `/poll/YEAR/WEEK` for each valid year/week combo. Extract voter usernames, ballot URLs, flair alt text, official/provisional classification. ~100 poll pages total.

**Phase 2 — Ballot pages:** Fetch `/ballot/ID` for each voter. Extract ordered 25-team list by display name. ~10,000+ ballot pages total across all seasons. Rate limit 0.3s between requests.

**Phase 3 — Resolve & generate:** Map display names → logo filenames, feed into `build_week_block()`.

**Storage:** One JSON file per season (`cbbpoll-YEAR.json`). Cache-aware — skip already-fetched ballots on re-run. Enables resumability for the ~15-20 min per season scrape time.

**JSON format:**
```json
{
  "year": 2022,
  "weeks": {
    "0": {
      "official": [
        {"username": "Phillyfan3", "flair_alt": "University of Virginia Logo",
         "ballot_id": 16037, "teams": ["UCLA", "Kansas", ...]}
      ],
      "provisional": [...]
    }
  }
}
```

## Working with this codebase

- After changing `style.css`, `script.js`, or HTML templates in `userpoll.py`, run `python3 userpoll.py --rebuild` to regenerate `index.html`, `ap-ballots.html`, and all fragment files
- Never hand-edit `index.html`, `ap-ballots.html`, `data-YEAR.html`, `data-ap-YEAR.html`, or `data-appage-YEAR.html` — all are fully generated
- Do not attempt to read `ballots.json` (too large)
- CSS lives in `style.css`, JS lives in `script.js` — both are read and inlined by `userpoll.py` at build time
- `DEBUG_LIMIT` global at top of `userpoll.py` can cap ballot fetches during development
- Team logos are referenced by team name only (e.g., `data-src="Duke"`) and resolved to `cbb-logos/Duke.webp` by JS
- Opening `index.html` as a local `file://` URL: current year works fine, but prior-year navigation and AP ballot loading won't work (fetch is blocked on file://). Use `python3 -m http.server 8000` and open `http://localhost:8000` for full local testing. Same applies to `ap-ballots.html`.

## JS performance architecture

Three caches in the JS reduce repeated DOM scanning:
- `_sortedTabsCache` — memoizes the sorted `.week-tab` array (stable after page load)
- `_leaderboardDataCache` — memoizes `buildLeaderboardData()` (all voter ballot data; never changes)
- `_deltaDataCache` / `_deltaDataCacheIsProvisional` — memoizes `buildDeltaData()`, invalidated when the Official/Provisional/Aggregate toggle changes

Year-split loading:
- `FRAGMENT_YEARS` array (emitted inline in `index.html`) lists userpoll years that need fetching; in `ap-ballots.html` this lists the AP page fragment years instead
- `AP_FRAGMENT_YEARS` array lists AP25 years that need fetching (only used in `index.html`)
- `loadYear(year)` fetches `${FRAG_PATH_PREFIX}${year}.html` — prefix defaults to `'data-'` on index.html; set to `'data-appage-'` on ap-ballots.html via injected constant
- `loadApYear(year)` fetches `data-ap-YEAR.html` — only used in `index.html`'s toggle flow
- Both inject into `#year-data-container`
- `_loadedYears` set tracks loaded years (AP years keyed as `ap-YEAR`); `_loadingYears` deduplicates concurrent fetches
- `showWeek()` is async — awaits `loadApYear()` when toggle is "AP Ballots"
- All year-navigation paths await `loadYear()` before calling `showWeek()`
- Background loader: current-year images first (16 concurrent), then all fragment files in parallel via `Promise.all`, then their images

### ap-ballots.html architecture

`ap-ballots.html` is a sibling of `index.html` generated by `build_ap_html()` / `rebuild_ap_html()` in `userpoll.py`. Key differences from `index.html`:
- AP weeks are the **primary** week tabs (no `data-poll` attribute, IDs like `week-YEAR-WEEKNUM`) — `getSortedTabs()` returns them (they pass the `data-poll !== "ap"` filter); `_apCurrentYear` stays null so all standard JS paths work directly
- Two injected JS constants: `const IS_AP_PAGE = true` and `const FRAG_PATH_PREFIX = 'data-appage-'`
- `FRAGMENT_YEARS` contains the AP page fragment years; `AP_FRAGMENT_YEARS` is empty
- Toggle text is "AP Ballots" with `cursor:default` (clicking is a no-op — `exitApBallots()` returns early when `_apCurrentYear === null`)
- Header has "← Userpoll" link instead of "AP Ballots →"
- Since CSS/JS are inlined identically, any feature added to the template or script automatically appears on both pages after `--rebuild`
