## Project Overview

Sing-box UI is a web-based configuration management tool for sing-box. It uses a Go backend with embedded Next.js frontend, containerized with Docker. The UI container manages a separate sing-box container via Docker socket (Docker-in-Docker pattern).

## Commands

### Frontend Development
```bash
cd frontend
npm install
npm run dev          # Dev server on :3000 (proxies API to :8080)
npm run build        # Build + copy output to ../server/dist
npm run lint         # Run ESLint
```

### Backend Development
```bash
cd server
go mod tidy
go run main.go       # Server on :8080
go test ./services -v  # Run tests
```

### Full Stack Development
Run both simultaneously:
- Terminal 1: `cd server && go run main.go`
- Terminal 2: `cd frontend && npm run dev`

### Docker Build
```bash
docker build -t singbox-ui .
docker-compose up -d   # Production deployment
```

## Architecture

### Build Flow
1. Frontend builds to static files (`next build` → `out/`)
2. Frontend output copied to `server/dist/`
3. Go embeds `dist/` directory at compile time via `go:embed`
4. Single binary serves both API and static frontend

### Key Directories
- `frontend/` - Next.js 16 + React 19 + TypeScript + shadcn/ui
- `frontend/components/` - Main UI components (inbound-config, outbound-config, routing-config, dns-config, subscription-manager, wireguard-*)
- `frontend/lib/api.ts` - API client (uses relative URLs)
- `frontend/lib/store/singbox-config.ts` - Zustand store for sing-box configuration
- `server/` - Go 1.24 + Gin framework
- `server/handlers/` - HTTP handlers
- `server/services/` - Business logic (docker, singbox, prober, subscription, wireguard, certificate)
- `server/dist/` - Embedded frontend (generated by build)

### Docker-in-Docker Pattern
The UI container manages sing-box via Docker socket:
- `HOST_DATA_DIR` env var provides host data path for sing-box container volume mounts
- UI container creates/manages `ghcr.io/sagernet/sing-box` container
- Config files stored in data directory, mounted into sing-box container
- Container command: `-D /var/lib/sing-box -C /etc/sing-box/ run`

### Prober Service
Async node health checking:
- Independent goroutines per node
- Thread-safe state with `sync.Map`
- Ring buffer for success rate calculation
- Auto-save/load from file

## API Structure

All API endpoints prefixed with `/api/`:
- `/api/singbox/*` - sing-box container lifecycle (config, run, stop, logs, status)
- `/api/wireguard/*` - Key generation, client config management
- `/api/subscription/*` - Subscription parsing and storage
- `/api/prober/*` - Node health check management
- `/health` - Health check endpoint

## Environment Variables

| Variable | Purpose |
|----------|---------|
| `DATA_DIR` | Container internal data directory (default: `/home/data`) |
| `HOST_DATA_DIR` | Host data directory for sing-box container mounts (default: `${PWD}/data`) |
| `LISTEN_ADDR` | Server listen address (default: `127.0.0.1:7000`) |
| `TZ` | Timezone (default: `Asia/Shanghai`) |

## sing-box Configuration Format

The frontend uses sing-box configuration format (not Xray-core format):

### Key Differences from Xray-core
- Route rules require `action` field (sing-box 1.11.0+)
- Use `rule_set` instead of deprecated `geosite`/`geoip`
- DNS servers have `type` field: "udp", "tcp", "https", "tls", "hosts"
- Outbound types: "direct", "block" (not "freedom", "blackhole")

### Outbound Tag Convention
All proxy outbound tags must use `proxy_out` consistently — single-node mode, balancer/urltest mode, DNS detour, and route final all reference this tag. Mismatched tags cause `outbound detour not found` errors.

### Route Rule Actions
- `route` - Route to outbound (default, requires `outbound` field)
- `sniff` - Protocol sniffing
- `hijack-dns` - Hijack DNS requests
- `reject` - Reject connection
- `resolve` - Resolve domain to IP

## Protocol Support

Inbound: WireGuard, Mixed, VLESS, VMess, Trojan, Shadowsocks, Hysteria2, TUIC, Naive, ShadowTLS, AnyTLS, HTTP
Outbound: All inbound protocols + direct + block
Subscription parsing: VMess, VLESS, Trojan, Shadowsocks, AnyTLS URLs + Clash YAML
TLS: ACME auto-cert (Let's Encrypt) + manual certificate upload
