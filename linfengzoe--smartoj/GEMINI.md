## 二、底层开发核心准则【强制遵守、无任何例外】

1. **编码规范**
   严格遵循《阿里巴巴 Java 开发手册》，后端采用 MVC/分层架构，前端遵循 Vue3 官方规范；无硬编码、无冗余代码，类/方法/变量命名见名知意。

2. **技术合规准则**
   仅使用稳定、主流、可落地的技术栈：

- 后端：Java17+、SpringBoot 2.x~3.x、SpringCloud、MyBatis/MyBatis-Plus、Redis、JWT、全局异常、事务控制
- 前端：Vue3、Vite、Element Plus、Axios
- 数据库：MySQL、合理索引、SQL 优化、防慢查询

3. **安全底层准则**
   默认规避 SQL 注入、XSS 攻击、跨域、权限漏洞；接口必做参数校验，后端必做全局异常捕获、事务控制。

4. **沙箱隔离与底层安全准则（针对动态代码执行）**
   绝对禁止使用 Java 原生的 `Runtime.getRuntime().exec()` 或 `ProcessBuilder` 直接在宿主机执行未知/不可信的外部代码。
   涉及用户代码评测与执行时，**必须强制使用 Docker 容器化技术（如引入 `docker-java` 库）**构建代码沙箱。
   沙箱生命周期必须严格受控：动态拉起 -> 注入代码与用例 -> 执行 -> **强制销毁（阅后即焚）**。
   必须在 API 层面强制进行资源隔离：严格限制容器的最大内存 (Memory Limit)、限制 CPU 周期，并**强制关闭容器网络 (Network Disabled)**，从根本上杜绝恶意代码、提权攻击和死循环耗尽资源。

5. **实用性准则**
   拒绝纸上谈兵，所有代码、配置、方案 **可运行**，附带依赖、配置、启动步骤，不输出残缺代码。

6. **注释强制准则**
   所有代码必须配备 **完整中文注释**，零基础可读，注释精准、无废话。

## 三、强制执行底层流程【不可跳过、不可颠倒】

收到任何 Java 全栈需求，严格按 5 步执行：

1. 需求底层解析：拆解业务、锁定范围、技术选型
2. 底层架构设计：分层结构、模块划分、数据表设计
3. 分模块开发：后端 → 配置 → SQL → 前端 → 工具类
4. 前置避坑：标注常见报错、坑点、解决方案
5. 验证优化：启动步骤、测试方法、性能优化
6. 根据需求动态裁剪流程：若是纯前端/纯后端需求，可跳过无关步骤，但整体逻辑推演步骤（解析->设计->开发->避坑->验证）不可跳过。
