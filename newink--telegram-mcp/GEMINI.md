# telegram-mcp — AGENTS.md

MCP server exposing Telegram via MTProto (userbot) as tools. Built with **Bun**, **TypeScript**, **mtcute**, **@modelcontextprotocol/sdk**.

Transport: Streamable HTTP on `/mcp`.

## Quick Start

```bash
bun install
cp .env.example .env       # fill in TELEGRAM_API_ID, TELEGRAM_API_HASH
bun auth                   # generates TELEGRAM_SESSION (requires real phone)
bun dev                    # starts MCP server on :3000/mcp
```

### Agent/Mock Mode (no real Telegram needed)

```bash
TELEGRAM_MOCK=true bun dev          # starts with mock data
TELEGRAM_MOCK=true bun test         # run tests
```

## Golden Principles

1. **MTProto is fragile.** Every API call can return FLOOD_WAIT. Always wrap in try/catch, never auto-retry.
2. **One TelegramClient per process.** Singleton in `telegram.ts`. Don't create a second client.
3. **Read-first.** Write operations (send/forward/delete) require whitelisting. Tests use mock only.
4. **Tools are atomic.** One tool = one operation. No combined "search and send" tools.
5. **bigint → string.** Telegram IDs are bigint. JSON can't serialize bigint. Always use `jsonResponse()` helper.
6. **Tests without real Telegram.** Use `TELEGRAM_MOCK=true`. Never call real API in tests.
7. **Write tools require config.** `delete_messages` and future write tools check `bot-data/config.yml` at each call. No config = disabled. See `src/config.ts`.

## Project Structure

```
src/
├── index.ts           # Entry point
├── server.ts          # MCP server, HTTP transport, tool registration
├── telegram.ts        # TelegramClient singleton (real or mock)
├── config.ts          # YAML config loader, write-tool restrictions
├── auth.ts            # QR-code auth flow
└── mock/
    ├── client.ts      # Mock TelegramClient implementation
    └── fixtures.ts    # Test data: chats, messages, media
```

## Environment Variables

| Variable | Required | Description |
|----------|----------|-------------|
| `TELEGRAM_API_ID` | Yes* | From https://my.telegram.org |
| `TELEGRAM_API_HASH` | Yes* | From https://my.telegram.org |
| `TELEGRAM_SESSION` | Yes* | Generated by `bun auth` |
| `PORT` | No | Server port (default: 3000) |
| `TELEGRAM_MOCK` | No | Set to `true` for mock mode |

*Not required when `TELEGRAM_MOCK=true`.

## Current Tools

- `search_dialogs` — Search dialogs by name/username
- `get_messages` — Get messages with date/unread filtering
- `search_messages` — Full-text search across chats
- `media_download` — Download media from a message
- `message_from_link` — Fetch message by t.me link
- `delete_messages` — Delete messages (requires config opt-in)
