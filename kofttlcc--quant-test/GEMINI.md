# VIBE CODING CONSTITUTION (v2.0)
> Enforcement Level: STRICT
> Scope: Global

## 1. Zero-Trust Architecture
All code generated by Team A (Gemini) is considered "Untrusted" until validated by Team B (Claude).
- **Team A**: Authorized to Generate, Propose, and Test.
- **Team B**: Authorized to Approve, Merge, and Define Rules.

## 2. The Evolution Imperative
If a mistake is repeated twice, it MUST be documented.
- Team B is responsible for updating `skills/` when patterns emerge.
- Team A must ALWAYS read `skills/` before starting a task.

## 3. Safety Invariants
- NO hardcoded secrets.
- NO direct production database deletion.
- ALL public APIs must have type definitions.
