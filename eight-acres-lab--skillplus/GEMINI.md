## What this is

`@e8s/skillplus` is a compilable skill package format + a TypeScript reference compiler + a curated catalog of community skills. The compiler turns a `.skillplus` directory into one of several target formats. It is deterministic and never calls a model.

## Conventions

- **Determinism over cleverness.** Same package + args â†’ byte-identical output. Provenance hashes depend on it.
- **No model calls.** The compiler produces prompts and contracts; running them is the runtime's job.
- **Minimal runtime deps.** Only `commander` (CLI) + `yaml` (parsing).
- **Scalar coercion.** `pythonCompatYAML` post-processes parsed values to coerce numbers to strings (preserves provenance hash parity with the original Python compiler).
- **Provenance always.** Every compiled output emits a `provenance_template`.
- **Targets are pure functions.** Adding a target = one branch in `renderTarget` + a `render<Name>` function.
- **`.search` file is mandatory** for any skill in `skills/` â€” the build-catalog script fails if missing or invalid.

## Adding a target

1. Add the name to `SUPPORTED_TARGETS` in `src/compile.ts`.
2. Document any new manifest metadata in `spec/`.
3. Implement `render<Name>(compiled): string`.
4. Wire into `renderTarget` dispatch.
5. Add a test case in `test/compile.test.ts`.

## Adding a vendor model profile

1. Create `model-profiles/<vendor>-<family>.yaml`.
2. Document prompt-style preferences, defaults, gotchas.
3. Verify reference packages compile cleanly against it.

## Adding a skill to the catalog

See [`CONTRIBUTING.md`](CONTRIBUTING.md). Short version: `skillplus init <id>`, fill TODOs, move into `skills/`, PR.
