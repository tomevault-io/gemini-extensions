## Development Environment

**IMPORTANT**: This project uses `uv` for package management. Direct `python` or `pip` commands will NOT work without activating the environment first.

### Running Commands

Always use one of these patterns:

```bash
# Pattern 1: Prefix with `uv run`
uv run python -c "from mcp_ticketer.cli.main import app; print('OK')"
uv run pytest tests/test_models.py -v

# Pattern 2: Activate shell first
source .venv/bin/activate
python -c "from mcp_ticketer.cli.main import app; print('OK')"
```

### Quick Commands

```bash
# Sync dependencies
uv sync

# Run tests (without coverage args that require pytest-cov)
uv run pytest tests/test_models.py -v -o addopts=""

# Import verification
uv run python -c "from mcp_ticketer.cli.main import app; print('OK')"

# Type checking
uv run mypy src/

# Compile check
uv run python -m py_compile src/mcp_ticketer/cli/main.py
```

### Common Errors

If you see `ModuleNotFoundError: No module named 'mcp_ticketer'`:

1. Run `uv sync` to ensure dependencies are installed
2. Use `uv run python ...` instead of `python ...`

If pytest fails with coverage/timeout plugin errors:

- Add `-o addopts=""` to bypass pytest.ini settings
