# claude-blog-cluster — Semantic Topic Cluster Engine

This is the development instructions file for the Semantic Cluster Engine
extension for claude-blog. It defines the architecture, conventions, and
development rules for the `blog-cluster` skill.

## Architecture


## Commands

| Command | Sub-Skill | Description |
|---------|-----------|-------------|
| `/blog cluster plan <keyword>` | `blog-cluster` | Semantic analysis → cluster plan + map |
| `/blog cluster plan --from strategy` | `blog-cluster` | Import blog-strategy output → cluster plan |
| `/blog cluster execute [plan]` | `blog-cluster` | Sequential blog-write with cluster context |

## Dependencies

This skill extends `claude-blog` and calls the following sub-skills during execution:
- `blog-write` — Writes each post in the cluster
- `blog-chart` — Called internally by blog-write for data visualizations
- `blog-seo-check` — Optional post-execution SEO validation

**No API keys required.** Uses Claude's built-in WebSearch + reasoning for semantic analysis.

## Development Rules

1. **Never modify claude-blog or claude-seo files** — This is a standalone extension
2. **Follow claude-blog SKILL.md conventions** exactly (frontmatter format, tool permissions)
3. **Use the ScienceExperts.ai brand palette** for all visualizations and output:
   - Primary accent: Signal Red `#D32F2F` (used sparingly—CTAs, emphasis, `.ai` suffix)
   - Text: White `#FFFFFF` (dark mode) / Black `#000000` (light mode)
   - Body: `#E5E7EB` (dark) / `#1F2937` (light, Charcoal)
   - Muted: Slate Gray `#6B7280` / `#9CA3AF`
   - Chart palette: `['#D32F2F', '#6B7280', '#EF5350', '#9CA3AF', '#374151']`
   - Never use: orange `#f97316`, sky blue `#38bdf8`, purple `#a78bfa`, green `#22c55e`
4. **References are loaded on demand** — Keep SKILL.md as the routing layer, details in references/
5. **Cluster context is the key innovation** — The metadata block passed to blog-write that makes posts cluster-aware and brand-consistent (includes voice, style, and CSS instructions)
6. **Enterprise quality = error handling** — Every edge case has a defined fallback behavior
7. **Deterministic execution order** — Same plan always produces same execution sequence
8. **Brand identity: ScienceExperts.ai** — All output carries ScienceExperts.ai attribution. Dark mode is the default. Inter is the body font. Signal Red is the single accent color.

## Output Files

|------|--------|-------------|
| `cluster-plan.json` | JSON | Plan Phase |
| `cluster-map.html` | HTML/SVG | Plan Phase |
| `cluster-scorecard.md` | Markdown | Execute Phase |
| `*.md` / `*.html` | Blog posts | Execute Phase (via blog-write) |

## Integration with claude-blog

The `blog-cluster` skill is designed to be installed alongside `claude-blog`.
When the user says `/blog cluster`, the blog orchestrator (in `skills/blog/SKILL.md`)
should route to this skill. If blog-cluster is installed, it seamlessly extends
the existing 17 commands with 2 new ones.

The install script handles the integration:
1. Copies `skills/blog-cluster/` to `~/.claude/skills/blog-cluster/`
2. That's it — Claude Code automatically discovers skills in `~/.claude/skills/`
