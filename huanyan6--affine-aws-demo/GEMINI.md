# CLAUDE.md — AFFiNE AWS Demo

## Project overview

Shell-script automation for self-hosting [AFFiNE](https://affine.pro) (open-source Notion alternative) on an AWS Free Tier EC2 instance. All infrastructure is created and torn down via five sequential scripts — no Terraform or CDK required.

## Repository layout

```
affine-aws-demo/
├── scripts/
│   ├── 01-aws-bootstrap.sh      # Create VPC, IAM, S3, key pair, Elastic IP
│   ├── 02-launch-ec2.sh         # Launch t2.micro, install Docker/Nginx via user-data
│   ├── 03-configure-affine.sh   # Deploy AFFiNE stack via SSH (runs locally)
│   ├── 04-test-affine.sh        # Automated test suite (runs locally, Docker checks via SSH)
│   └── 05-teardown.sh           # Tag-based sweep — destroys ALL project resources
├── config/                      # Git-ignored; holds bootstrap-outputs.env at runtime
├── cis/                         # CIS v2.0 security verification tools (Prowler, Lynis, docker-bench)
├── docs/                        # Deployment and CIS verification guides
└── CLAUDE.md                    # This file
```

## AWS resources created

| Resource | Detail |
|---|---|
| VPC | 10.10.0.0/16, DNS enabled |
| Public subnet | 10.10.1.0/24 |
| Internet Gateway + route table | Default route to IGW |
| Security group | TCP 80/443 open, SSH restricted to runner's IP |
| EC2 instance | t2.micro, Amazon Linux 2023, 20 GB gp3 |
| Elastic IP | Static public address |
| SSH key pair | Saved to `~/.ssh/affine-demo-key.pem` |
| IAM user + group + policy | S3 read/write for AFFiNE file uploads |
| S3 bucket | `affine-demo-files-<account-id>`, public access blocked |

Estimated cost: **~$0/month** within AWS Free Tier (first 12 months).

## Prerequisites

- WSL (Ubuntu/Kali) or macOS/Linux
- AWS CLI configured (`aws sts get-caller-identity` works)
- `jq`, `openssl`, `curl`, `ssh`, `scp` available in shell
- GitHub account (for pulling `ghcr.io/toeverything/affine:stable`)

## Deployment — step by step

```bash
git clone https://github.com/huanyan6/affine-aws-demo.git
cd affine-aws-demo
chmod +x scripts/*.sh

./scripts/01-aws-bootstrap.sh   # ~2 min  — prompts for local + EC2 usernames once
./scripts/02-launch-ec2.sh      # ~5 min  — waits for instance status checks
./scripts/03-configure-affine.sh # ~10 min — waits for user-data, deploys via SSH
./scripts/04-test-affine.sh     # ~1 min  -- validates deployment
```

When finished:
```bash
./scripts/05-teardown.sh        # removes ALL AWS resources for this project
```

## Key design decisions

### Usernames set once in script 01
`01-aws-bootstrap.sh` prompts for:
- **Local username** (e.g. `huan_y` in WSL) — used for the SSH key path
- **EC2 OS username** (default `ec2-user` for Amazon Linux 2023) — used in user-data and SSH commands

Both are saved to `config/bootstrap-outputs.env` and reused by all subsequent scripts automatically.

### Script 03 runs locally, not on EC2
`03-configure-affine.sh` orchestrates everything from the local machine via SSH:
1. Polls SSH until the instance accepts connections
2. Polls `/var/log/affine-setup.log` (with `sudo`) until user-data completes
3. SCPs files to `~/` first, then `sudo mv` to `/opt/affine/` (avoids permission errors)
4. Generates and pushes `.env` and `docker-compose.yml` via SSH heredoc
5. Starts Nginx, runs `docker compose up`, waits for AFFiNE on port 3010

### Script 04 runs locally, Docker checks via SSH
`04-test-affine.sh` runs HTTP/S3 tests locally and SSH-proxies Docker checks (container health, PostgreSQL, Redis) to EC2 — avoids the WSL docker-stub issue.

### Script 05 uses tag-based discovery
`05-teardown.sh` queries AWS by tag/name pattern rather than reading IDs from the env file. This means **all duplicates** from repeated runs are cleaned up, not just the last recorded set.

### AFFiNE Docker image
Uses the official public image: `ghcr.io/toeverything/affine:stable`
- `ghcr.io/toeverything/affine-graphql:stable` is now private — do not use
- The `affine_migration` service runs DB migrations before the main container starts

## config/bootstrap-outputs.env

Auto-generated by `01-aws-bootstrap.sh` and appended to by `02-launch-ec2.sh`. Git-ignored (contains AWS credentials). Key variables:

```
LOCAL_USER, LOCAL_HOME     # local machine username and home dir
EC2_USER                   # EC2 OS username (e.g. ec2-user)
KEY_FILE                   # absolute path to SSH .pem file
EIP, EIP_ALLOC             # Elastic IP address and allocation ID
INSTANCE_ID                # EC2 instance ID
S3_BUCKET                  # S3 bucket name
AFFINE_AWS_ACCESS_KEY/SECRET_KEY  # IAM credentials for AFFiNE S3 access
```

## Common issues and fixes

| Symptom | Fix |
|---|---|
| `InvalidKeyPair.NotFound` on script 02 | Teardown deleted the AWS key but local `.pem` remained — script 01 now detects and recreates both |
| `scp: Permission denied` on `/opt/affine/` | Old script ran on EC2 before user-data finished; script 03 now SCPs to `~/` and `sudo mv`s |
| `curl` conflicts on `dnf install` | AL2023 ships `curl-minimal`; script 02 no longer installs `curl` separately |
| `unauthorized` / `denied` on `ghcr.io` | Image moved — now using `ghcr.io/toeverything/affine:stable` |
| Teardown only removes one instance | Script 05 now uses tag-based AWS queries to find and delete all duplicates |
| SSH post-quantum warning | `LogLevel=ERROR` added to SSH_OPTS in scripts 03 and 04 |
