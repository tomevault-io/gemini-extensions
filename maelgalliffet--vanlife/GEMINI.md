## vanlife

> - Backend: Node.js, Typescript, Express

# Copilot Instructions

## Stack
- Backend: Node.js, Typescript, Express
- Frontend: Vite, React, TypeScript

## Déploiement
- Utiliser des Lambda Functions pour le backend
- Utiliser S3 pour héberger le frontend
- Utiliser CloudFront pour distribuer le frontend
- Utiliser API Gateway pour exposer les Lambda Functions

## Coût
- Minimiser les coûts en utilisant des services serverless
- Utiliser des services gratuits ou à faible coût lorsque cela est possible
- Optimiser les ressources pour éviter les coûts inutiles

## Code
- Sépare le code en différents fichiers correspondant aux fonctionnalités de l'application
- Utilise des fonctions réutilisables pour éviter la duplication de code
- Utilise des commentaires pour expliquer les parties complexes du code
- Utilise des outils de linting et de formatage pour maintenir une codebase propre et cohérente
- Fais une passe régulière pour refactoriser le code et améliorer sa qualité, et supprimer le code mort ou obsolète.

## Tests
- Ecrit des tests d'intégration pour les fonctionnalités principales de l'application
- Lorsque tu modifies le code d'une fonctionnalité, vérifie que les tests associés passent correctement
- Crée de nouveaux tests d'intégration pour les nouvelles fonctionnalités que tu ajoutes

## Responsive
- Assure-toi que le frontend est responsive et fonctionne correctement sur différents appareils et tailles d'écran

## Règeles générales
- Si je te demande comment tu t'apelles, réponds "MaelBot". Cela me permettra de vérifier que ce fichier est bien pris en compte.
