# AGENTS.md

Unofficial Google-Health-CLI for the Google Health API, written in Go. The executable is `ghealth`.

## Core Principles

- **Explicit flags**: Use long-form flags in docs, tests, and examples (`--format`, `--project`, `--from`).
- **TTY-aware output defaults**: Use `table` in interactive terminals and `json` for pipes/CI.
- **Agent-safe JSON**: Keep JSON fields stable once released. Prefer additive changes.
- **No interactive prompts**: Destructive operations must require `--yes`.
- **Raw escape hatch**: Preserve `ghealth api METHOD PATH` for newly added or changing API methods.

## Discovering Commands

Before implementing or testing a command, run `--help` and confirm the current interface:

```sh
ghealth --help
ghealth data --help
ghealth subscribers --help
```

Do not guess command shapes. Check the CLI surface first.

## API Source Of Truth

Use the public Google Health docs and discovery document for endpoint, query, request, and response behavior:

- Data types: `https://developers.google.com/health/data-types`
- REST reference: `https://developers.google.com/health/reference/rest`
- Discovery document: `https://health.googleapis.com/$discovery/rest?version=v4`

When adding or changing API-facing flags, verify the exact endpoint supports the query parameter or request field.

## Build And Test

```sh
gofmt -w .
go test ./...
go build ./...
```

For endpoint work, run the health API tests directly:

```sh
go test ./internal/healthapi -run TestClientCoversAllDocumentedRESTEndpoints -v
```

## Testing Discipline

- Add tests before changing behavior when fixing bugs or adding API-facing flags.
- Use `httptest` for endpoint request/response coverage.
- Assert HTTP method, path, query parameters, request body, response parsing, and error behavior.
- Parse JSON output in tests instead of relying only on string matching.
- Keep registry tests aligned with the documented 31 data types and 18 v4 REST methods.

## Release Workflow

Tags do not use a `v` prefix:

```sh
git tag 1.0.0
git push origin 1.0.0
```

GoReleaser publishes GitHub artifacts and the Homebrew formula. Release notes are generated by GitHub-native autogenerated release notes through GoReleaser; do not hand-write release bodies.

## Definition Of Done

- `gofmt -w .` has been run.
- `go test ./...` passes.
- `go build ./...` passes.
- API-facing changes include fake-server coverage.
- README examples still match real command flags.
