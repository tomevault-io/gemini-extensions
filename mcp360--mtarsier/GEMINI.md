# mTarsier — Agent Guide

> Read this before making any changes to the codebase.

## Project Overview

mTarsier is a desktop app for managing MCP (Model Context Protocol) servers across all AI clients. Built with Tauri v2 (Rust backend) + React 19 + TypeScript + Tailwind CSS v4 + Zustand.

## Key Paths

| Path | Purpose |
|---|---|
| `src/` | React frontend |
| `src/pages/` | Full-page views (Dashboard, Clients, Config, Marketplace, Settings, About) |
| `src/components/` | Shared UI components |
| `src/components/marketplace/` | Marketplace-specific components |
| `src/store/` | Zustand state stores |
| `src/lib/` | Utilities (clients registry, themes, utils) |
| `src/types/` | TypeScript types |
| `src-tauri/src/` | Rust backend |
| `src-tauri/src/commands/` | Tauri commands (config, clients, audit, cli, io, utils) |
| `src-tauri/src/bin/tsr.rs` | CLI binary entry point |
| `src-tauri/scripts/` | Build helper scripts |
| `src-tauri/binaries/` | Compiled sidecar binaries (gitignored, generated by prepare-sidecar.sh) |

## Architecture

```
React UI  ──invoke()──▶  Tauri Commands (Rust)  ──▶  Filesystem / OS
                                │
                                ▼
                         app_lib (shared Rust lib)
                                │
                                ▼
                         tsr CLI binary (clap)
```

## Frontend Conventions

- **Styling**: Tailwind CSS v4 with custom design tokens in `src/index.css`
- **State**: Zustand stores in `src/store/` — `clientStore`, `configStore`, `themeStore`, `settingsStore`
- **Tauri calls**: Always use `invoke<T>("command_name", { args })` with try/catch
- **Themes**: Colors accessed via CSS variables (`text-primary`, `bg-surface`, `text-text-muted`, etc.)
- **Component size**: Keep under 150 lines — extract if larger

## Design Tokens

| Token | Value | Usage |
|---|---|---|
| `primary` | `#00ff88` | Accent, active states, success |
| `cyan` | `#00d4ff` | Secondary accent |
| `amber` | `#ffb800` | Warnings, hints |
| `base` | `#080c0a` | Page background |
| `surface` | slightly lighter | Cards, sidebar |
| `border` | subtle | Borders |
| `text-muted` | muted | Secondary text |

## Rust Backend Conventions

- No `unwrap()` in production code — use `?` operator or `map_err`
- All Tauri commands return `Result<T, String>`
- File paths: always expand `~` using `dirs` crate, handle macOS/Windows/Linux
- `commands/mod.rs` — register all modules here
- `lib.rs` — register all commands in `invoke_handler`

## Client Registry

All supported MCP clients are defined in `src/lib/clients.ts` as `CLIENT_REGISTRY`. Each entry has:
- `id`, `name`, `type` (Desktop/IDE/CLI/Web)
- `configPath` (macOS), `configPathWin`, `configPathLinux`
- `configKey` — JSON key where `mcpServers` live
- `configFormat` — `json` or `toml`
- `detection` — how to detect if installed (`app_bundle`, `cli_binary`, `vscode_extension`, `none`)
- `supportedTransports` — `stdio`, `sse`, `streamable-http`, `remote-mcp`

## CLI (tsr)

- Entry point: `src-tauri/src/bin/tsr.rs`
- Uses `clap` with derive macros
- Commands are currently stubbed — implementations go in `src-tauri/src/commands/cli.rs`
- Does NOT use Tauri — keep it free of Tauri imports to avoid binary bloat

## Build & Dev

```bash
# Install dependencies
pnpm install

# Prepare CLI sidecar (run once before dev)
cd src-tauri && bash scripts/prepare-sidecar.sh && cd ..

# Dev mode
pnpm tauri dev

# Production build (macOS, signed)
APPLE_SIGNING_IDENTITY="Developer ID Application: ..." pnpm release:signed
```

## CI/CD

- Workflow: `.github/workflows/release.yml`
- Triggers on `git tag v*`
- Builds for: macOS (aarch64 + x86_64), Windows (NSIS), Linux (AppImage + deb + rpm)
- macOS builds are signed and notarized automatically via secrets

## Git Conventions

### Branch Naming

Always use the format `<type>/<short-description>` with lowercase kebab-case:

| Prefix | When to use | Example |
|---|---|---|
| `feature/` | New functionality | `feature/marketplace-search` |
| `fix/` | Bug fixes | `fix/client-detection-crash` |
| `perf/` | Performance improvements | `perf/async-client-detection` |
| `refactor/` | Code restructuring, no behavior change | `refactor/extract-auth-section` |
| `chore/` | Tooling, deps, config, CI | `chore/update-tauri-2.5` |
| `docs/` | Documentation only | `docs/cli-usage-guide` |
| `test/` | Tests only | `test/client-store-unit` |
| `release/` | Release prep / version bumps | `release/v1.1.0` |

### Commit Messages

Follow [Conventional Commits](https://www.conventionalcommits.org/):
```
<type>(<optional scope>): <short description>

feat(marketplace): add server search filtering
fix(clients): prevent concurrent detection calls
perf(clients): offload detection to spawn_blocking
chore: update Cargo dependencies
```

## Common Pitfalls

- `src-tauri/binaries/` is gitignored — run `prepare-sidecar.sh` after cloning
- Claude Code config uses dual-scope: check both `mcpServers` and `projects.{HOME}.mcpServers`
- When adding a new Tauri command: register it in both `commands/mod.rs` and `lib.rs`
- Tailwind v4 uses CSS variables for theming — do not use hardcoded hex colors in components
