# CLAUDE.md — SoftMig Project Context

## What this project is

SoftMig is a SLURM-integrated software GPU slicing layer for shared NVIDIA GPU
clusters. It intercepts CUDA and NVML calls via `LD_PRELOAD` to enforce per-job
memory limits and SM compute throttling. Based on HAMi-core, adapted for
Alliance/Compute Canada HPC environments. CUDA 12+ only.

## Build

```bash
./build.sh              # CMake Release build, outputs build/libsoftmig.so
CMAKE_OPTIONS="-DCMAKE_BUILD_TYPE=Debug" ./build.sh  # debug build
```

Install target and update procedures are in `docs/BUILD_AND_INSTALL.md`.

## Source layout

```
src/
  libsoftmig.c          — dlsym interposition hook, CUDA/NVML dispatch, init
  dlsym_resolve.c       — shared helper to find real dlsym (dlvsym fallbacks)
  nvml_cache.c          — per-device TTL cache for NVML process lists
  process_utils.c       — UID lookup, cgroup session matching
  utils.c               — PID detection, file locking, CUDA_VISIBLE_DEVICES
  log_file.c            — file logging under /var/log/softmig
  allocator/            — GPU memory chunk tracker with OOM enforcement
  cuda/                 — cuMemAlloc, cuMemcpy, cuLaunchKernel hooks, etc.
  nvml/
    hook.c              — NVML library loader, dispatch table, GetMemoryInfo hook
    nvml_entry.c        — NVML function wrappers with cgroup/UID process filtering
  multiprocess/
    config_file.c       — reads /var/run/softmig/{jobid}.conf (prolog-written)
    multiprocess_memory_limit.c  — shared region, OOM killer, per-device limits
    multiprocess_utilization_watcher.c  — SM throttle watchdog thread
    shrreg_tool.c       — CLI to inspect shared memory regions
  include/              — headers (nvml-subset.h is the NVML ABI definitions)
docs/examples/          — prolog, epilog, install scripts for SLURM
test/                   — test runners and CUDA probe binaries
```

## How interposition works

1. `libsoftmig.so` is loaded via `/etc/ld.so.preload` or `LD_PRELOAD`.
2. It exports a `dlsym()` wrapper that intercepts every `cu*` and `nvml*`
   symbol lookup, returning SoftMig's hooked versions.
3. A linker version script (`softmig.ver`, auto-generated by
   `gen_version_script.sh`) ensures wrappers are visible in the dynamic symbol
   table for apps that link `-lcuda` directly.
4. On first `cuInit`, the real CUDA/NVML libraries are loaded via `dlopen`,
   the shared memory region is initialized, and the utilization watcher starts.
5. `config_file.c` reads limits from root-owned files in `/var/run/softmig/`
   written by the SLURM prolog. In SLURM jobs, environment variable fallback
   is disabled — config files are the sole source of truth.

## Key data flow

- **Memory enforcement**: `cuMemAlloc` → allocator checks summed NVML usage
  against limit → OOM killer if exceeded. NVML usage is summed per-process
  using cgroup/UID filtering, with a 1-second TTL cache.
- **Process isolation**: `nvmlDeviceGetComputeRunningProcesses` hooks filter
  the process list by cgroup session (SLURM job) or UID fallback.
- **SM throttling**: utilization watcher thread sleeps kernel launches when
  a job exceeds its SM utilization quota.

## Key constraints

- Config files must be root-owned, regular files (no symlinks) — enforced by
  `config_file.c` with `lstat()` + ownership checks.
- Shared region lock uses `flock()` (not `O_EXCL` create/delete).
- Array jobs: config lookup falls back from `{jobid}_{arrayid}.conf` to
  `{jobid}.conf` since `SLURM_ARRAY_TASK_ID` may not be set in prolog.
- `nvmlProcessInfo_t` is defined locally in `nvml-subset.h` to match the
  driver ABI — do not include system `<nvml.h>` for struct definitions.
