## general

> General rules for the project


## Code

- do not commit changes by default, always check with me if you can commit

## New features:

- read README.md for reference
- use similar structure for new features
- always add specs for new features / new code

## Ruby style guide:

- make sure code is compatible with "rubocop", you can run "rubocop -A" to fix most issues
- always check rubocop issues

## Documentation:

- always include documentation in README.md
- update content in "Features section" with new features
