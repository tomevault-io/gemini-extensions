# Agent guide for portbook

You are working on **portbook**, a Rust CLI + local web UI that lists
the listening ports on a developer's machine. Read the right doc for
the question you're answering — don't guess from code alone.

## Map of source-of-truth docs

| If you need to know…                                | Read                                                    |
|-----------------------------------------------------|---------------------------------------------------------|
| What portbook does, how to use it                   | [README.md](README.md)                                  |
| How the code is layered (Engine / CLI / UI)         | [ARCHITECTURE.md](ARCHITECTURE.md)                      |
| What good code looks like here (CLI quality bar)    | [CONVENTIONS.md](CONVENTIONS.md)                        |
| **How to commit and release** (process rules)       | [CONTRIBUTING.md](CONTRIBUTING.md)                      |
| What's planned, what's deferred                     | [BACKLOG.md](BACKLOG.md)                                |
| How release notes are generated                     | [cliff.toml](cliff.toml) + [CONTRIBUTING.md](CONTRIBUTING.md#release-flow) |

When these conflict with code, the doc wins — fix the code or update
the doc in the same change. Don't let drift accumulate.

## Hard rules

1. **Every commit follows [Conventional
   Commits](https://www.conventionalcommits.org/).** Non-conforming
   commits are dropped from release notes. Full type table, scopes,
   and examples in [CONTRIBUTING.md](CONTRIBUTING.md#commit-conventions).
2. **Never edit `CHANGELOG.md` by hand** — it's regenerated by
   `git cliff` on every release. Fix the commit message instead.
3. **Match the CLI quality bar** in
   [CONVENTIONS.md](CONVENTIONS.md) before adding flags or
   subcommands. Look for the standard convention before inventing one.
4. **Stay inside the layering** described in
   [ARCHITECTURE.md](ARCHITECTURE.md). Engine doesn't know about
   views; CLI and web UI are peers.

## Releasing

See [CONTRIBUTING.md → Release flow](CONTRIBUTING.md#release-flow).
Short version: bump `Cargo.toml`, `cargo build`,
`git cliff --tag vX.Y.Z -o CHANGELOG.md`, commit as
`chore(release): vX.Y.Z`, tag, push.
