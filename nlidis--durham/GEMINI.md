# Website Overhaul - Guidance for Claude

## Project Overview
We are taking an old website from a small owner and changing it to improve it. I want to make the website look more professional, using the same information they have but make it more concise and organized better.

## Tech Stack
- **Languages:** html, python
- **Database:** not necessary

## Project Structure
- `/src/components`: UI components
- `/src/lib`: Shared utility functions
- `/docs`: External documentation references

## Development Commands
- **Dev:** `npm run dev`
- **Test:** `npm test`
- **Lint:** `npm run lint`

## Critical Rules
- Never delete `config/` files without explicit confirmation.
- reference the small company's old website to get all of the information and images using the mcp server: https://tracklements.com/
- use another mcp server to see what you have built
- make the page look less cluttered while keeping all of the same information
- use the @looks folder for the images I want you to use in the website and the logos/aestetics1
- make sure all information added was explicitly said in the other website, don't halucinate facts2
