# Instructions

Project: Chat with agents on Telegram through ACP.


## 0x00. Importance

All changes generated by Large Language Models (LLMs) MUST be reviewed and approved by a **human** before being committed. The commit message MUST include an explicit confirmation of this review.

Example:
```text
**Add Documents**

This commit generated by AI. All changes authored by me.

<On Codex, By GPT 5.4>
Prompt:
Your job is trying to add documents in this swift package project.
- You should use Apple DocC format;
- You should add README.md;
- You should add doc for every swift file in @/Sources  ;
```


## 0x01. Tech Stack

- Python 3.14
- Package Management: [uv](https://docs.astral.sh/uv/)
- CLI Framework: [Typer](https://typer.tiangolo.com)
- Async Runtime: [asyncio](https://docs.python.org/3/library/asyncio.html)
- Core Framework: 
  - [telethon](https://docs.telethon.dev/en/stable/)
  - [acp](https://github.com/agentclientprotocol/python-sdk)
  - [mcp](https://github.com/modelcontextprotocol/python-sdk)
- Other Devlopment Framework: 
  - tests: pytest + pytest-asyncio; 
  - lint/format: ruff; 
  - types: pyright + ty;


## 0x02. Source code

- `README.md` the basic inforamtion about this project.
- `AGENTS.md` the instructions for agents like codex.
- `docs` the sub doc file, the most them are used for agents.
- `packages` the packages:
  - `tele_acp_core` the data model and protocol 
  - `telegram_channel` the channel implementation for telegram
- `src/tele_acp` the source code 
  - `cli` the cli entrypoint
  - `app` the application bootstrap
  - `acp` the implementation of the ACP client protocol
  - `mcp` the local MCP server exposed to ACP agents
  - `channel` the channel hub where contonls all message income and outcome
  - `chat` the chat
  - `replier` the replier that reply to the income message.
  - `config` the applicatoin configuration settings
  - `utils` the output, formatting, and helper utilities
